<objective>
Lay the data-layer foundation for Phase 10's three new exercise types and the premium quota gate. Extend the database schema with three new accuracy columns on `user_song_progress`, add a new `user_exercise_song_counters` table tracking per-user-per-family song set membership, evolve `deriveStars` from `0|1|2` to `0|1|2|3`, add `deriveBonusBadge`, widen the `ExerciseType` union with the three new values (`grammar_conjugation`, `listening_drill`, `sentence_order`), add `RATING_WEIGHTS` entries, extend the `EXERCISE_FEATURE_FLAGS` map with a new `song_quota` gate status, and extend `checkExerciseAccess` to accept a `songVersionId` option and consult the per-family counter.

Purpose: Every other Phase 10 plan consumes these primitives. Without the union, the schema columns, the counter, and the gate signature change, the exercise plans cannot dispatch, write accuracy, or enforce quotas.
Output: Drizzle migration + schema + feature flags + access gate + counters module + FSRS weights, with unit tests for pure functions.
</objective>
