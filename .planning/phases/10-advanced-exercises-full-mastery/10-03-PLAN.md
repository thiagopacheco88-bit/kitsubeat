    2. **Conjugation module** `src/lib/exercises/conjugation.ts`:
       - Import `parseConjugationPath`, `StructuredConjugation` from `scripts/lib/conjugation-audit`.
       - Export `V1_CONJUGATION_FORMS` (from audit).
       - Export `pickConjugationOptions(params): { correct: string; distractors: string[]; base: string; form: string } | null`:
         - Inputs: `targetVocab: VocabEntry`, `grammarPoint: GrammarPoint`, `sameJlptPool: VocabEntry[]` (verb filter applied by caller).
         - Parse `grammarPoint.conjugation_path`. If not structured OR form not in `V1_CONJUGATION_FORMS` → return `null`.
         - Correct answer = `parsed.conjugated`.
         - Same-verb wrong conjugation: use a tiny rule table to generate the OPPOSITE-polarity or ADJACENT-form (e.g., if target form = past_affirmative, wrong = past_negative; if te_form, wrong = negative). Keep ≤ 40 LOC.
         - Same-JLPT alternate verbs: pick 2 distinct verbs from `sameJlptPool` (filter by part_of_speech === "verb"), conjugate each to target form via the same mini rule-table. Handle irregular verbs (する → した/して/しない, くる → きた/きて/こない) as hardcoded cases.
         - De-dup distractors (no collision with correct or each other; trim+lowercase per phase 08-02 decision).
         - If pool insufficient (< 2 eligible verbs), return partial with filler `"???"` OR return `null` — prefer null to avoid garbage options.

    3. **Generator branch** `src/lib/exercises/generator.ts`:
       - Add case in `makeQuestion` per RESEARCH.md Example 2 (extended):
         ```ts
         case "grammar_conjugation": {
           const gp = vocab.grammar_point; // or however the vocab→grammar link is resolved
           const parsed = parseConjugationPath(gp?.conjugation_path);
           if (!parsed || !parsed.is_structured) return null;
           if (!V1_CONJUGATION_FORMS.includes(parsed.conjugation_type)) return null;
           const ref = findVerseForVocab(parsed.conjugated, verses);
           if (!ref) return null;
           const verse = verses.find((v) => v.verse_number === ref.verseNumber)!;
           const verseText = verse.tokens.map((t) => t.surface).join("");
           const opts = pickConjugationOptions({ targetVocab: vocab, grammarPoint: gp, sameJlptPool });
           if (!opts) return null;
           return {
             id: ...,
             type: "grammar_conjugation",
             prompt: verseText.replace(parsed.conjugated, "_____"),
             correctAnswer: opts.correct,
             options: shuffle([opts.correct, ...opts.distractors]),
             conjugationBase: opts.base, // e.g., "食べる"
             verseRef: ref,
           };
         }
         ```
       - In `buildQuestions`, when mode includes grammar_conjugation: skip songs whose parseConjugationPath yields zero structured paths (CONTEXT: "Skip those songs entirely").

    4. **Tests** `src/lib/exercises/__tests__/conjugation.test.ts`:

       - Unstructured path → `null`.
       - Empty same-JLPT pool → `null` (no garbage filler).
       - Irregular verb する/くる → correctly conjugated.
       - Distractors never include the correct answer.
