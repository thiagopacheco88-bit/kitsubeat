    - Export `async function checkVocabIdentity(db, slug, version, lesson): Promise<Gap[]>`
    - For every `lesson.vocabulary[i].vocab_item_id`:
      1. Assert it exists and is a valid UUID (regex)
      2. Query `SELECT id FROM vocabulary_items WHERE id = $1` — if not found, emit Gap `{ field: vocabulary[i].vocab_item_id, type: 'invalid', detail: 'orphaned UUID' }`
    - Also emit a Gap when a vocab entry has NO vocab_item_id and the row's lesson is non-null
    - Batch-query the vocabulary_items table once per song (IN (...)) to avoid N+1

    Edit scripts/seed/06-qa-agent.ts:
    - Import `checkVocabIdentity` and call it inside the per-version loop; merge its gaps into the main gaps array
    - Add a furigana completeness check inline: for every token where `token.surface` contains kanji (regex `/[\u4e00-\u9fff]/`), assert `token.reading` is non-empty AND contains hiragana/katakana (`/[\u3040-\u30ff]/`). Emit Gap with field `verse[N].tokens[i].reading` on violation.
    - Add TV-pack skip at the top of the per-version loop: if `songVersion.lesson === null` AND (slug matches TV pattern or song has a metadata flag identifying TV-pack — use the simplest heuristic that works today; document in comment), increment a `skippedTv` counter and `continue` without emitting gaps.
    - At end of run, log: `Skipped ${skippedTv} TV-pack versions awaiting WhisperX lesson generation.` When `--json` mode is on, include `skipped_tv_versions: N` in the output.
    - Ensure exit code behavior unchanged: skipped rows never count toward gaps.
  </action>
