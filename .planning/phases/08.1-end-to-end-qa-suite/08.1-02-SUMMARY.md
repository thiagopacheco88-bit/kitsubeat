## Decisions Made

- **Determinism test wording:** the plan offered a fallback for the case where the generator is non-deterministic (Math.random shuffle). Implemented the fallback path: assert length-stability and per-type-bounds across 5 runs rather than exact-ordering. When generator.ts grows a seeded shuffle, this test becomes the gate that catches accidental de-seeding.
- **Test B encoded as `it.fails` (not deleted):** the plan explicitly suggested `it.fails` for the thin-pool gap. Kept it as an executable TODO so the regression flips RED automatically the moment the generator gains a thin-pool fallback (the most likely Phase 8 follow-up). Documented in the test header so a future contributor knows to convert `it.fails` → `it` and delete the comment block when the fix lands.
- **Premium-list test handles vacuous-truth explicitly:** Phase 8 ships zero premium types today. Rather than skip the test or assert false-positively, the test asserts `PREMIUM_TYPES === []` and returns early — making the contract loud and re-enabling automatically when Phase 10 adds premium types.
- **deriveStars test imports from `@/lib/db/schema`** (not `@/lib/db` or `@/lib/db/index`) — schema.ts holds pure drizzle table defs and the `deriveStars` function with no DB connection. Verified with `grep` that no test file imports `@/lib/db/index.ts`, satisfying the plan's no-DB-in-unit-layer constraint.
- **Test-DB-isolation invariants verified by grep, not new code:** the plan's verification ("No test file imports from src/lib/db/index.ts" and "No vi.mock/jest.mock") was confirmed via grep on `src/**/__tests__/**` — both return zero matches.

## Known Gaps (Phase 8 Follow-up)

- **Thin-pool 3-distractor fallback** — When a song has the minimum 3 vocab entries AND the JLPT pool is empty, `buildQuestions` currently emits questions with 2 distractors (not 3). The most likely fix paths:
  1. Recycle existing distractors (allow duplicates with shuffled positions) — cheapest but reduces distractor diversity
  2. Fall back to adjacent-JLPT-level pools (e.g. an N5 song with no N5 pool falls back to N4 / N6 entries)
  3. Hard-coded common-word fallback (e.g. embed top-100 N5 vocab as a static fallback)
  - **Test gate:** `distractor-picker.test.ts` Test B (`it.fails`) — flips RED automatically the moment any of these are wired.
