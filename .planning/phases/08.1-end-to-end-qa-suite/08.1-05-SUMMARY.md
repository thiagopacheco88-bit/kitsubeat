## Decisions Made

- **Real YouTube iframe, no postMessage stubs.** Plan + CONTEXT both lock this. Stubbing the YT API would mask race conditions in YouTubeEmbed.tsx's onStateChange + tracking interval — the exact class of bug the E2E layer exists to catch. Cost: tests skip when YouTube is unreachable (geo-restricted, network out, ad-block in CI). Documented as graceful fallback.
- **Single-condition gate (`=== 'test'`) for ALL test-only hooks.** Never OR'd with NODE_ENV. The same pattern is used by the pre-existing `src/stores/exerciseSession.ts` exposure of `window.__kbExerciseStore`. OR'ing with `dev` would leak instrumentation into every developer machine; OR'ing with NODE_ENV would leak into preview environments.
- **`data-verse-number` and `data-active` are UNCONDITIONAL.** They're tiny, useful in dev devtools, and don't expose any sensitive timing data. Only `data-start-ms` (which leaks the raw WhisperX timestamps) sits behind the test-env gate.
- **1500ms sync floor (not 250ms).** 250ms is the perception target measured in isolation; 1500ms is what survives real network jitter on CI hardware. Tightening this without a separate performance plan would re-introduce flake. The plan's objective explicitly calls this out.
- **Seek-past-end test accepts EITHER outcome** (last verse stays active OR no verse active). Both are defensible product behaviors; the spec should anchor reality, not aspirations. If the team changes the behavior, the test is updated alongside the change.
- **`.first()` everywhere in lesson-toggle spec** — GrammarSection ALSO renders its own per-card "Romaji ON/OFF" toggle. Without `.first()`, toggle assertions ambiguously match either the player controls OR the grammar card's button. Documented inline.
- **Translation language test asserts presence, not text.** Not every song has PT-BR translations seeded; the lesson schema falls back to `en` silently. The test asserts that translation paragraphs render (any `.text-gray-300` is visible), not that the text is literally Portuguese.
- **Token popup integrity = "contains the · separator + non-empty content".** The middle dot is rendered between reading and romaji in TokenPopup.tsx — its presence proves the popup mounted with both fields populated, without coupling to specific reading strings that might change.
- **Verse-3 NOT active assertion in backward-seek test.** Stronger than just "verse 1 active" — proves the active highlight ACTUALLY moved instead of duplicating. Catches a class of regression where the active class accumulates instead of swapping.
