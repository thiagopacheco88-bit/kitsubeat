## Decisions Made

See `key-decisions` in the frontmatter above. Top three for skim-readers:

1. **Quarantine override via env-driven sentinel, not `--grep-invert ''`.** Playwright AND-combines grep + grepInvert and rejects empty `--grep-invert`. The only way to opt quarantined tests INTO a run is to neutralize the config-level invert. Implementation: a literal sentinel pattern that can never appear in a test title (`__never_match_kb_quarantine_sentinel__`). When KB_RUN_QUARANTINE=1, grepInvert flips to that sentinel and the CLI `--grep '\[kb-quarantine\]'` filter alone determines what runs.
2. **Fastest-first layer order in `measure-suite-runtime.ts`.** test:qa (~30s) -> test:unit (~10s) -> test:integration (~30s) -> test:e2e (~8min). A regression in any of the three fast layers kills the run before E2E starts the dev server — that alone saves ~10 minutes per failed run on the slow layer.
3. **PR job skips E2E entirely.** `test:ci-pr` runs qa + unit + integration only. Browser install + Playwright execution + iframe network is the bulk of suite time AND the bulk of CI flake risk. Nightly is the only place where E2E + the 15-min budget gate live. Speed/cost tradeoff is intentional and documented in the workflow comment block.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Quarantine grep-invert override required env-var-driven sentinel, not the inline `playwright --grep` script the plan suggested**
- **Found during:** Task 2 verification (scratch spec to confirm quarantine is filtered by default + included by `npm run test:quarantine`).
- **Issue:** The plan's package.json suggested `"test:quarantine": "playwright test --grep '\\[kb-quarantine\\]'"`. But Playwright AND-combines `grep` (CLI) and `grepInvert` (config) — a test must match grep AND NOT match grepInvert. With the config's `grepInvert: /\[kb-quarantine\]/`, the CLI `--grep '\[kb-quarantine\]'` matches 0 tests because every quarantined test is excluded by grepInvert before the grep filter runs. There is no `--grep-invert ''` escape hatch (Playwright validates that the value is a non-empty regex).
- **Fix:** Made `grepInvert` env-var-driven in playwright.config.ts: defaults to `/\[kb-quarantine\]/`, flips to a never-match sentinel `/__never_match_kb_quarantine_sentinel__/` when `KB_RUN_QUARANTINE=1`. Wrote `scripts/qa/run-quarantined.ts` to set the env var and forward to `playwright test --grep '\[kb-quarantine\]'`. Updated package.json to `"test:quarantine": "tsx scripts/qa/run-quarantined.ts"`.
