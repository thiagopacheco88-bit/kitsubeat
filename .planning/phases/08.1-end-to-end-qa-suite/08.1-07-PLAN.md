CONTEXT (08.1) Coverage Scope locks "Geo-restricted / missing YouTube videos fail gracefully" — which requires the graceful-failure behavior to EXIST before we can assert it. The new user-features prohibition applies to NEW screens/exercises; a minimal inline error state on an existing component is NOT a new feature — it is the observable behavior CONTEXT coverage requires. Adding it is in scope.

    **Step 1 — Add minimal fallback UI to src/app/songs/[slug]/components/YouTubeEmbed.tsx:**
    - Add a `const [embedState, setEmbedState] = useState<'loading' | 'ready' | 'error'>('loading')` at the top of the component.
    - In the YT.Player config, add an `onError` event handler that calls `setEmbedState('error')`. YouTube iframe errors surface as events with numeric codes (100 = not found / 101/150 = embedding disabled). Treat any error code as 'error'.
    - In `onReady`, call `setEmbedState('ready')`.
    - Add a watchdog timer: 15s after mount, if `embedState === 'loading'`, call `setEmbedState('error')` (covers cases where the iframe never loads at all due to network / route blocking — the YT API never fires onReady or onError).
    - If `embedState === 'error'`, render an inline fallback block INSTEAD of the container div:
      ```tsx
      <div data-yt-state="error" className="aspect-video w-full flex items-center justify-center rounded-lg bg-neutral-900 text-neutral-300 p-6 text-center">
        <div>
          <p className="font-medium">Video unavailable</p>
          <p className="mt-1 text-sm opacity-80">This video may be geo-restricted or removed. The lyrics and lesson content below still work.</p>
        </div>
      </div>
      ```
    - If `embedState === 'loading'`, keep the existing container div but add `data-yt-state="loading"` to it. If `embedState === 'ready'`, add `data-yt-state="ready"`.
    - Do NOT add any new dependency. Do NOT move the component file.
    - Copy is fixed: "Video unavailable" (heading) + "This video may be geo-restricted or removed. The lyrics and lesson content below still work." (body). No i18n keys needed — the rest of the player UI is not translated yet.

    **Step 2 — Create tests/e2e/regression-geo-fallback.spec.ts:**

    - Option B: use a song with a known-gone youtube_id (seed one into TEST_DATABASE_URL under the 3rd SEEDED_SLUG — `broken-video-test` with youtube_id like `__geo-blocked-xxx` that YouTube rejects). Document the seeded fixture in tests/support/seed-test-db.ts.
