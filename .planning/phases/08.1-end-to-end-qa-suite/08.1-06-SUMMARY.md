## user_vocab_mastery wiring (Task 3)

`saveSessionResults` Step 7 (lines 154–254 of `src/app/actions/exercises.ts`):

1. Filter `answers[]` to unique `vocabItemId` values (Set-based dedup)
2. For each unique vocab id:
   - Aggregate accuracy across this session's answers for that vocab (any-correct heuristic — rewards mastery, not single mistakes)
   - Compute FSRS rating via `ratingFor(answer.type, anyCorrect, {})` from `src/lib/fsrs/rating.ts`
   - SELECT existing mastery row (may be null for first-encounter vocab)
   - Compute next FSRS schedule via `scheduleReview(prev, rating, intensity)` from `src/lib/fsrs/scheduler.ts`
   - UPSERT into `user_vocab_mastery` with `onConflictDoUpdate` on `(user_id, vocab_item_id)`
3. Per-vocab failures isolated via try/catch — a single mastery write cannot fail the song-progress save

`grep -n "user_vocab_mastery\|userVocabMastery" src/app/actions/exercises.ts` returns 13 matches across the import line, AnswerRecord doc, comments, and the four FROM/INSERT/UPDATE/conflict-target SQL fragments.

The `ts-fsrs` package was already installed (^5.3.2) and is used via the 08.2-01 scheduler wrapper — no new dependencies added.

## Decisions Made

- **Gate is single-condition.** `process.env.NEXT_PUBLIC_APP_ENV === 'test'` — no `||` with `NODE_ENV`, no secondary fallback. Next.js inlines `NEXT_PUBLIC_*` at build time so this evaluates to `false` in the production bundle and the assignment is dead code, tree-shaken away. The store reference never reaches the client in production.
- **No `data-correct` attribute.** The plan explicitly forbids exposing the correct answer as a data-attribute. The window hook is the ONLY way the test reads the answer — and it's gated.
- **Reuse 08.2-01 FSRS scheduler instead of inlining a heuristic.** `ts-fsrs` is already in package.json. The 08.2-01 plan built `src/lib/fsrs/scheduler.ts::scheduleReview` and `src/lib/fsrs/rating.ts::ratingFor` exactly for this purpose. No reason to duplicate the logic.
- **AnswerRecord.vocabItemId is OPTIONAL.** Backward-compat with any caller that hasn't been updated. The Set-based dedup in Step 7 already filters out answers without it.
- **Test cleanup queries `userId='anonymous'`.** The production UI (`SongContent.tsx` line 186) hardcodes `userId="anonymous"` until Clerk auth lands. Querying for the actual userId being written is the only honest assertion. When auth lands, BOTH the UI hardcode AND the test cleanup switch together.
- **Star 2 confetti banner is the canonical assertion** (canvas count is best-effort warn-only). canvas-confetti unmounts quickly; the `"You earned Star 2!"` banner is a stable structural assertion.
- **Repeat-session test exits via `Practice Again` CTA, not `page.reload()`.** Exercises the production retry flow (`onRetry` → `clearSession` → config screen → Start). Reload would skip the clearSession step.
- **`playwright.config.ts` testMatch=*.spec.ts.** Without it, Playwright was trying to collect `tests/integration/*.test.ts` (vitest tests) and crashing on `@vitest/expect`. Defensive guard at the runner level is cleaner than directory-based.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Added `testMatch=*.spec.ts` and `testIgnore=**/integration/**` to playwright.config.ts**

- **Found during:** Task 3 verification (`npx playwright test --list`)
- **Issue:** Playwright was trying to collect `tests/integration/*.test.ts` (the vitest tests authored by plan 08.1-03), which import from `vitest` and crash with `Cannot redefine property: Symbol($$jest-matchers-object)` at collect time. `--list` showed 0 tests.
- **Fix:** Added `testMatch=["**/*.spec.ts"]` (matches Playwright convention) and `testIgnore=["**/support/**", "**/integration/**"]` (defensive — also matches if someone names a vitest file with `.spec.ts`).
- **Files modified:** `playwright.config.ts`
- **Verification:** After the fix, `npx playwright test --list --reporter=line` lists 29 tests across `app.spec.ts` + 6 e2e specs (including the 4 new exercise specs and the 2 from plan 05).
- **Committed in:** `082c815`

**2. [Rule 3 - Blocking] Set `webServer.env.NEXT_PUBLIC_APP_ENV=test` in playwright.config.ts**

- **Found during:** Task 1 implementation (the test-only hook needs the build flag)
- **Issue:** The plan added the `window.__kbExerciseStore` gate but the gate evaluates to `false` unless `NEXT_PUBLIC_APP_ENV=test` is set when Next.js builds the bundle. `npm run dev` would not have set it, leaving `__kbExerciseStore === undefined` and all exercise specs would hang at `waitForFunction`.
- **Fix:** Playwright `webServer.env: { NEXT_PUBLIC_APP_ENV: "test" }` ensures the dev-server child process inherits the flag.
- **Files modified:** `playwright.config.ts`
- **Note:** A linter merged my comment with a fuller multi-hook note covering plan 05's `__kbPlayer` and `data-start-ms` instrumentation — kept verbatim.
- **Committed in:** `d782df3`

**3. [Rule 2 - Missing Critical] AnswerRecord.vocabItemId left OPTIONAL (not required)**

- **Found during:** Task 3 saveSessionResults extension
- **Issue:** Plan said "Extend AnswerRecord to include vocabItemId: string". Making it required would break any in-flight legacy caller (the recordVocabAnswer plan 08.2-02 just landed and calls things slightly differently).
- **Fix:** Added as `vocabItemId?: string`. Step 7 filters out missing values via the Set-based dedup. Behaviour: legacy callers continue to write user_song_progress; new callers also write user_vocab_mastery.
- **Files modified:** `src/app/actions/exercises.ts`
- **Verification:** TypeScript compiles cleanly across the touched files.

**4. [Rule 3 - Blocking] Used `'anonymous'` userId for DB cleanup in exercise-progress-fsrs.spec.ts**

The plan's user-id assumption would assert against zero rows and false-fail.
- **Fix:** The FSRS spec defines `PROD_HARDCODED_USER_ID = "anonymous"` and queries/cleans that user. Documented prominently in the spec header and in this summary.
- **Files modified:** `tests/e2e/exercise-progress-fsrs.spec.ts`
- **Operator follow-up:** When Clerk auth lands, BOTH the UI hardcode AND the test query switch together.
