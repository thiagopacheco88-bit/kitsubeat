## Files Created/Modified

- `scripts/seed/06-qa-uuid-integrity.ts` — exports `checkVocabIdentity(db, slug, version, lesson)` returning `UuidGap[]`. Three failure modes: missing UUID, invalid UUID syntax, orphaned UUID (not in vocabulary_items). Direct-execution guard at the bottom calls `runStandalone()` so `tsx scripts/seed/06-qa-uuid-integrity.ts` works.
- `scripts/seed/06-qa-geo-check.ts` — probes every `song_versions.youtube_id`, batches 50 per `videos.list` call, p-limit 8 concurrent batches. Three exit classes (OK / GEO / GONE) with rich per-row context. Flags: `--allow-geo` (downgrade GEO to warning), `--json` (structured output), `--verbose` (print OK lines).
- `scripts/seed/06-qa-agent.ts` — now imports `checkVocabIdentity`; `validateVersion` is async and merges UUID gaps into the report; `validateToken` adds the kanji-surface => kana-reading furigana check; main loop skips `version_type='tv' && lesson===null` and counts them in `versions_skipped_tv` / `skipped_tv_versions`. New summary line: "Skipped N TV-pack versions awaiting WhisperX lesson generation."
- `package.json` — added `test:qa:uuid` and `test:qa:geo` scripts (both run via tsx with the scripts tsconfig).

## Decisions Made

- **TV-pack heuristic** = `version_type='tv' AND lesson IS NULL`. Matches the project context (60 TV rows awaiting WhisperX) exactly without needing a new schema field. Documented in the script comment for future maintainers.
- **No `scripts/lib/youtube-availability.ts`.** The plan permitted extracting probe helpers there if `backfill-geo-check.ts` didn't already expose them. They were already exported from `scripts/lib/youtube-search.ts` (functions `fetchVideosMetadata`, `classifyAvailability`, constant `AMERICAS_REGIONS`, error `YouTubeQuotaExceededError`). Both scripts now import from that single file — one source of truth, no extra indirection.
- **`test:qa:uuid` runs the file directly** rather than wrapping with `tsx -e 'import(...).then(...)'`. The standalone module ends with a `process.argv[1]?.endsWith("06-qa-uuid-integrity.ts")` guard that calls `runStandalone()`. Cleaner npm script line and easier to copy/paste.
- **`tier === "americas"` reported as OK, not GEO.** A video that's blocked outside AMERICAS_REGIONS but still embeddable for users in the Americas is functionally available for the target audience. Only `tier === "restricted"` (no overlap with target regions, or non-embeddable, or non-public) becomes GEO.
- **GONE always fails, even with `--allow-geo`.** A deleted/private/non-embeddable video is a hard catalog defect — there's no operator scenario where ignoring it makes sense.

## Deviations from Plan

None - plan executed exactly as written. The plan asked us to "use whichever is cleanest" for `test:qa:uuid` between (a) wrapping with `tsx -e` and (b) adding a `main()` to the standalone module. Option (b) was picked and documented above; this is a planned choice point, not a deviation.

The plan also said "If backfill-geo-check.ts doesn't expose a helper, extract the probe function into scripts/lib/youtube-availability.ts and import it from BOTH places." It DOES expose them (via `scripts/lib/youtube-search.ts` which both scripts already import from), so no extraction was needed — the single-source-of-truth invariant is satisfied. Documented in the script header.
