<feature>
  <name>Exercise Question Generator</name>
  <files>src/lib/exercises/generator.ts, src/lib/exercises/__tests__/generator.test.ts</files>
  <behavior>
  The generator is a pure function with no side effects or network calls.

  **Types:**
  - ExerciseType: "vocab_meaning" | "meaning_vocab" | "reading_match" | "fill_lyric"
  - Question: { id, type, vocabItemId, prompt, correctAnswer, distractors (3), explanation, detailedExplanation?, verseRef? }
  - SessionConfig: { mode: "short" | "full", targetCount: 10 for short, all*4 capped at 40 for full }

  **buildQuestions(lesson, mode, jlptPool) -> Question[]:**
  - Filters vocabulary to entries with vocab_item_id (skip entries without UUID)
  - For each vocab entry, generates one question per exercise type (4 types)
  - For fill_lyric: only generates if a verse exists with start_time_ms > 0 containing the word's surface form
  - Shuffles all questions using Fisher-Yates
  - Short mode: returns first 10 (or fewer if total < 10)
  - Full mode: returns all, capped at 40
  - Questions with fewer than 3 vocab items in song: disable fill_lyric (can't make 4 unique options)

  **pickDistractors(correct, type, sameSongPool, jlptPool) -> string[]:**
  - Returns exactly 3 distractor strings
  - First draws from same-song pool (excluding correct answer)
  - If < 3 candidates, pads from jlptPool (excluding correct)
  - If still < 3, pads from jlptPool ignoring JLPT level (adjacent levels)
  - Deduplicates: no distractor matches correctAnswer after trim + lowercase
  - No duplicate distractors in the returned array

  **Extracting the right field per exercise type:**
  - vocab_meaning: prompt=surface, correctAnswer=meaning (localized to "en"), distractors=other meanings
  - meaning_vocab: prompt=meaning, correctAnswer=surface, distractors=other surfaces
  - reading_match: prompt=surface, correctAnswer=romaji, distractors=other romaji
  - fill_lyric: prompt=verse text with surface blanked, correctAnswer=surface, distractors=other surfaces

  **Test cases (input -> expected output):**
  - 5 vocab entries, short mode -> 10 questions (5 * 4 = 20, sliced to 10)
  - 5 vocab entries, full mode -> 20 questions (5 * 4 = 20, under cap)
  - 12 vocab entries, full mode -> 40 questions (12 * 4 = 48, capped to 40)
  - 2 vocab entries -> no fill_lyric questions (< 3 entries, can't make 4 options)
  - Vocab entry without vocab_item_id -> skipped entirely
  - All questions have exactly 3 distractors
  - No distractor equals the correct answer (case-insensitive, trimmed)
  - Fill-the-lyric only generated when verse has start_time_ms > 0
  - Vocab entry with no matching verse -> no fill_lyric question for that entry
  </behavior>
  <implementation>
  Implement in `src/lib/exercises/generator.ts`:

  1. Fisher-Yates shuffle (unbiased): standard implementation, NOT arr.sort(() => Math.random() - 0.5)
  2. extractField(vocab, type): returns the answer string based on exercise type. Use `localize(vocab.meaning, "en")` for meaning fields.
  3. pickDistractors(correct, type, sameSongPool, jlptPool): implements the same-song-first, JLPT-fallback strategy with deduplication
  4. findVerseForVocab(surface, verses): finds first verse containing the surface form and having start_time_ms > 0. Returns { verseNumber, startMs } or null.
  5. buildQuestions(lesson, mode, jlptPool): orchestrates the full generation pipeline
  6. makeQuestion(vocab, type, distractors, verses): creates a single Question object with UUID id (use crypto.randomUUID()), explanation text

  For explanations: generate simple template strings like "「{surface}」means '{meaning}'" for vocab_meaning, "'{meaning}' is written as 「{surface}」" for meaning_vocab, etc. These are 1-2 sentence teacher-like explanations per locked decision.

  For detailedExplanation: include the example_from_song field and additional_examples[0] if available.

  Install vitest if not already present (check package.json first). Use `vitest` for tests.
  </implementation>
</feature>
