**Server action (src/app/actions/exercises.ts):**

"use server" module with `saveSessionResults` function. Follows research Pattern 9 exactly.

Input: `{ userId: string, songVersionId: string, answers: Array<{ questionId: string, type: ExerciseType, chosen: string, correct: boolean, timeMs: number }>, mode: "short" | "full", durationMs: number }`

Logic:
1. Separate answers into ex1_2_3 types (vocab_meaning, meaning_vocab, reading_match) and ex4 type (fill_lyric)
2. Compute accuracy for each group: correct count / total count. If a group has 0 answers, accuracy is null.
3. Completion increment: short = +15, full = +30
4. Upsert user_song_progress using Drizzle `onConflictDoUpdate`:
   - completion_pct: `LEAST(100, current + increment)`
   - ex1_2_3_best_accuracy: `GREATEST(COALESCE(current, 0), new_accuracy)` (only update if better)
   - ex4_best_accuracy: same GREATEST pattern
   - sessions_completed: current + 1
   - updated_at: NOW()
5. After upsert, SELECT the updated row to get current stars via `deriveStars()`
6. Return `{ completionPct, stars, previousStars }` so UI can detect star earn events

Add `// TODO: replace with Clerk userId` comment where userId is used.

**Progress queries (add to src/lib/db/queries.ts):**

Add two functions:

1. `getUserSongProgress(userId, songVersionId)` — returns single row or null from user_song_progress

2. `getUserSongProgressBatch(userId, songVersionIds[])` — returns Map<songVersionId, progress> using single `WHERE song_version_id IN (...)` query. This prevents N+1 on the browse page (Pitfall 6 from research).

Both functions return the raw row plus `stars` computed via `deriveStars()`.

3. Wait for response to get `{ completionPct, stars, previousStars }`
4. If stars > previousStars, trigger StarDisplay confetti (pass `newStarEarned: true`)

- Large accuracy percentage in center (green if >= 80%, yellow if >= 60%, red if < 60%)
