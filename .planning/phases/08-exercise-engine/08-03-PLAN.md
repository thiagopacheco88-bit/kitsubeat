**Zustand store (src/stores/exerciseSession.ts):**

Create the exercise session store following research Pattern 2 exactly:

```typescript
import { create } from "zustand";
import { persist, createJSONStorage } from "zustand/middleware";
```

State shape:
- songVersionId: string | null
- questions: Question[] (from generator.ts types)
- currentIndex: number
- answers: Record<string, { chosen: string; correct: boolean; timeMs: number }>
- startedAt: number | null
- mode: "short" | "full" | null
- _hasHydrated: boolean

Actions:
- startSession(songVersionId, questions, mode): resets all state, sets startedAt to Date.now()
- recordAnswer(questionId, chosen, correct, timeMs): adds to answers record
- advanceQuestion(): increments currentIndex
- clearSession(): resets all to initial values
- setHasHydrated(v): sets _hasHydrated

Persist config:
- name: "kitsubeat-exercise-session"
- storage: createJSONStorage(() => localStorage)
- onRehydrateStorage: sets _hasHydrated to true via callback

**CRITICAL (Pitfall 2):** Export a helper `isSessionForSong(store, songVersionId)` that returns true only if store.songVersionId matches AND store.questions.length > 0 AND store.currentIndex < store.questions.length. ExerciseTab must call clearSession() when this returns false.

**JLPT pool route (src/app/api/exercises/jlpt-pool/route.ts):**

Create a GET route that accepts `?jlpt_level=N5` query param. Queries `vocab_global` materialized view filtered by jlpt_level, joins vocabulary_items to get surface/reading/romaji/meaning/part_of_speech, returns up to 50 entries as JSON. This gives the exercise generator its distractor pool for same-JLPT-level words not in the current song.

Use the existing `db` import from `@/lib/db`. Query vocab_global via raw SQL since it's a materialized view:
```sql
SELECT vi.id, vi.dictionary_form, vi.reading, vi.romaji, vi.part_of_speech, vi.meaning
FROM vocab_global vg
JOIN vocabulary_items vi ON vi.id = vg.vocab_item_id
WHERE vg.jlpt_level = $1
GROUP BY vi.id, vi.dictionary_form, vi.reading, vi.romaji, vi.part_of_speech, vi.meaning
LIMIT 50
```

Return 200 with JSON array. Return 400 if jlpt_level param is missing.

**ExerciseTab.tsx — Config screen + session gate:**

Props: `{ lesson: Lesson, songVersionId: string }`

On mount:
1. Read `_hasHydrated` from useExerciseSession. If false, render a skeleton/loading state.
2. Once hydrated, check `isSessionForSong(store, songVersionId)`. If true (existing session matches), jump directly to ExerciseSession (resume).
3. If no matching session, show config screen:
   - Title: "Practice"
   - Two mode cards: "Quick Practice" (10 questions, ~2 min) and "Full Lesson" (all vocab, ~5-8 min)
   - Each card shows estimated question count and time
   - "Start" button fetches JLPT pool from `/api/exercises/jlpt-pool?jlpt_level={lesson.jlpt_level}`, calls buildQuestions(lesson, mode, jlptPool), then store.startSession(songVersionId, questions, mode)

State machine: config -> session -> (summary handled by Plan 04)

**ExerciseSession.tsx — Question loop orchestrator:**

Props: none (reads from Zustand store)

Renders:
1. Top progress bar: filled proportionally (currentIndex / questions.length). Use a Tailwind div with dynamic width.
2. Question counter below bar: "{currentIndex + 1}/{questions.length}"
3. Current QuestionCard for questions[currentIndex]
4. On answer recorded + advance, if currentIndex >= questions.length, set a `sessionComplete` flag (stored in component state, not Zustand — summary will read final answers from store)

300ms CSS transition between questions (opacity fade via Tailwind `transition-opacity duration-300`).

**QuestionCard.tsx — Single question with 4 answer options:**

Props: `{ question: Question, onAnswered: (chosen: string, correct: boolean, timeMs: number) => void }`

Renders:
1. Question prompt text (large, white)
2. For fill_lyric type: show verse text with the target word replaced by "____"
3. Four answer option buttons in a 2x2 grid or vertical stack
4. Shuffle the 4 options (correctAnswer + 3 distractors) ONCE on mount (useMemo), not on every render
5. On click: record start time, compute timeMs, determine correct, call onAnswered
6. After click: disable all buttons, show FeedbackPanel inline below the options

Answer option states:
- idle: default gray border
- correct (green): `border-green-500 bg-green-500/10`
- incorrect (red): `border-red-500 bg-red-500/10`
- neutral (other options after answer): slightly dimmed

When wrong answer is selected: highlight the selected option in red AND the correct option in green (per locked decision: "Wrong answers show both the correct answer AND an explanation of why the user's choice was wrong").

**FeedbackPanel.tsx — Inline feedback + "More" detail panel:**

Two modes:
1. **Inline (default):** Appears below the answer options after selection. Shows:
   - Green checkmark or red X icon
   - 1-2 sentence explanation from question.explanation
   - If wrong: "The correct answer is: {correctAnswer}" + explanation of why user's choice was wrong (template: "「{chosen}」means '{chosen_meaning}' — not what was asked here.")
   - Small "More" button (text link style)
   - "Continue" button (primary, calls onContinue)

2. **Full-screen detail panel (triggered by "More"):** Fixed overlay with:
   - Question prompt repeated
   - Full detailed explanation from question.detailedExplanation
   - Example from song (question.explanation includes this)
   - "Continue" button that closes panel AND advances to next question

Tone: warm and educational per locked decision. Example: "Great job! 「食べる」(taberu) means 'to eat' — it's one of the most common verbs you'll encounter."
