  <action>
1. Install: `npm install kuroshiro @sglkc/kuromoji`. Note: use `@sglkc/kuromoji` (not vanilla `kuromoji`) to avoid Node 18+ async dictionary loading failures per RESEARCH.md anti-pattern.

2. Create `scripts/lib/lrclib.ts` following RESEARCH.md Pattern 3 and the LRCLIB code example:
   - `fetchFromLrclib(title: string, artist: string, anime: string)`: GET `https://lrclib.net/api/get?track_name=...&artist_name=...&album_name=...`. Returns `{ synced: string | null, plain: string } | null`.
   - `parseLrc(lrc: string)`: Parse LRC `[mm:ss.xx]` format into `Array<{ startMs: number; text: string }>`. Handle both 2-digit and 3-digit centisecond fields.
   - No API key required. No rate limiting needed for 200 calls.

3. Create `scripts/lib/genius.ts`:
   - `searchGenius(title: string, artist: string)`: GET `https://api.genius.com/search?q={title} {artist}` with `Authorization: Bearer {GENIUS_API_KEY}`. Return the first hit's URL.
   - `fetchGeniusLyrics(url: string)`: Fetch the Genius song page HTML, extract lyrics text from the page. Use a simple regex or HTML parser to extract lyrics from the `data-lyrics-container` divs. Return plain text lyrics (no timestamps).
   - Note: Genius returns unsynced plain text only.

4. Create `scripts/lib/kuroshiro-tokenizer.ts` following RESEARCH.md kuroshiro code example:
   - Initialize kuroshiro with `@sglkc/kuromoji` analyzer. Dict path: `node_modules/@sglkc/kuromoji/dict`.
   - `initKuroshiro()`: async init, called once at script start. Cache the instance.
   - `tokenizeLyrics(lyrics: string)`: tokenize via `kuroshiro._analyzer.tokenize(lyrics)`. For each token with non-empty surface_form, produce `{ surface, reading (hiragana), romaji (hepburn), pos (kuromoji POS tag) }`.
   - For romaji: use `kuroshiro.convert(token.surface_form, { to: "romaji", mode: "okurigana", romajiSystem: "hepburn" })`.
   - Filter out whitespace-only tokens.
  </action>

  </verify>

1. Create `scripts/seed/02-fetch-lyrics.ts`:
   - Load dotenv from `.env.local`.
   - Read `data/songs-manifest.json` via SongManifestSchema validation.
   - Initialize kuroshiro once at startup.
   - For each song in manifest:
     a. Check if `data/lyrics-cache/{slug}.json` already exists (checkpoint/resume — skip if cached).
     b. Try LRCLIB first (`fetchFromLrclib(title, artist, anime)`).
     c. If no LRCLIB match, try Genius (`searchGenius(title, artist)` then `fetchGeniusLyrics(url)`).
     d. If no Genius match, mark as `source: "pending_whisper"` with empty lyrics (Plan 03 timing pipeline will handle transcription).
     e. If lyrics found: run `tokenizeLyrics(lyrics)` to get pre-computed tokens with readings and romaji.
     f. Write to `data/lyrics-cache/{slug}.json`: `{ slug, title, artist, source: "lrclib"|"genius"|"pending_whisper", raw_lyrics: string, synced_lrc: string|null, tokens: TokenArray }`.
   - Use p-limit(2) for concurrent fetches (LRCLIB has no rate limit, but be polite).
   - At end, print summary: `X from LRCLIB, Y from Genius, Z pending Whisper`.
   - Run via: `npx tsx scripts/seed/02-fetch-lyrics.ts`.

2. Important: Lyrics are cached as raw text for the Claude prompt. They are NOT stored verbatim in the final database — Claude generates grammatical breakdowns per the copyright mitigation decision. The raw lyrics serve only as input context for content generation.
