<task type="auto">
  <name>Task 1: Build the Verse Coverage Agent (QA script)</name>
  <files>
    scripts/seed/06-qa-agent.ts
  </files>
  <action>
1. Create `scripts/seed/06-qa-agent.ts` following RESEARCH.md Pattern 6 but with expanded coverage checks:
   - Load dotenv, initialize DB connection.
   - Query all songs from the database.
   - For each song, validate:

   **Song-level checks:**
   - `lesson` JSONB is not null
   - `jlpt_level` is set
   - `difficulty_tier` is set
   - `youtube_id` is not empty
   - `timing_data` is not null (warn if timing_verified is still "auto")
   - `content_schema_version` matches current version (1)

   **Lesson-level checks (if lesson exists):**
   - `verses` array is non-empty
   - `vocabulary` array is non-empty
   - `grammar_points` array is non-empty

   **Per-verse checks:**
   - `tokens` array is non-empty
   - `translations` object has keys "en", "pt-BR", "es" (all three required)
   - Each translation is a non-empty string
   - `literal_meaning` is non-empty
   - `start_time_ms` and `end_time_ms` are > 0 (timing populated)

   **Per-token checks:**
   - `surface` is non-empty
   - `reading` is non-empty (furigana)
   - `romaji` is non-empty
   - `grammar` is a valid enum value
   - `grammar_color` is a valid enum value
   - `meaning` is non-empty

   **Per-vocabulary checks:**
   - `part_of_speech` is a valid enum value
   - `jlpt_level` is set
   - `example_from_song` is non-empty

   - Collect all gaps as `{ slug: string, field: string, type: "missing"|"empty"|"invalid"|"outdated_schema" }`.
   - At the end, print a structured JSON report: `{ total_songs, songs_complete, songs_with_gaps, gaps_count, gaps: Gap[], timing_status: { auto: N, reviewed: N, approved: N } }`.
   - Exit with code 0 if zero gaps, code 1 if any gaps exist.
   - Add `--verbose` flag that prints per-song details instead of just the summary.
   - Add `--json` flag that outputs raw JSON for programmatic consumption.
   - Run via: `npx tsx scripts/seed/06-qa-agent.ts`.
  </action>
  <verify>Run `npx tsx scripts/seed/06-qa-agent.ts` against the database. Verify the output shows the gap report with total songs, complete count, and gap details. If gaps exist, verify the exit code is 1. If no gaps, exit code is 0.</verify>

Reports gaps as structured JSON and exits with code 1 if any gaps found.</done>
