   - `LessonSchema`: jlpt_level (N5-N1), difficulty_tier (basic/intermediate/advanced), verses array, vocabulary array, grammar_points array.

<task type="auto">
  <name>Task 2: Build the 200-song manifest builder CLI script</name>
  <files>

    scripts/lib/anidb-themes.ts

  <action>
1. Install: `npm install p-limit`.

2. Create `scripts/lib/jikan.ts`:
   - Function `fetchTopAnimeByPopularity(page: number)`: calls Jikan v4 `GET /top/anime?filter=bypopularity&page={page}&limit=25`. Returns array of `{ mal_id, title, title_english }`.
   - Function `fetchAnimeThemes(malId: number)`: calls Jikan v4 `GET /anime/{malId}/themes`. Returns `{ openings: string[], endings: string[] }`. Each string is format like `"1: \"Song Title\" by Artist Name (eps 1-25)"`.
   - Function `parseThemeString(theme: string)`: regex parser that extracts `{ number, title, artist }` from the Jikan theme string format. Handle edge cases: quoted titles, multiple artists with "feat." or "&", parenthetical episode ranges. Return null if unparseable.
   - Rate limit all Jikan calls to 3 req/s using p-limit(3) with a 350ms delay between calls.

3. Create `scripts/lib/spotify-charts.ts` (per locked decision: Spotify is a required source):
   - Function `getSpotifyAccessToken()`: uses Client Credentials flow (`POST https://accounts.spotify.com/api/token` with `SPOTIFY_CLIENT_ID` and `SPOTIFY_CLIENT_SECRET`). Cache the token for its TTL.
   - Function `fetchSpotifyAnimeThemes()`: search Spotify for well-known anime theme playlists (e.g., search query "anime openings", "anime OP ED", "anime themes") via `GET /v1/search?q=anime+openings&type=playlist&limit=5`. For each top playlist, fetch its tracks via `GET /v1/playlists/{id}/tracks`.
   - Extract `{ title, artist, spotify_popularity }` from each track. Deduplicate by `title + artist`.
   - Return an array of `{ title, artist, spotify_rank }` sorted by descending Spotify popularity score.
   - Rate limit: Spotify allows 30 req/s, use p-limit(5) to be polite.

4. Create `scripts/lib/anidb-themes.ts` (per locked decision: AniDB is a required source):
   - Function `fetchAnidbThemes()`: fetch the AniDB anime themes dump from their public API or the community-maintained `anime-offline-database` GitHub resource (https://raw.githubusercontent.com/manami-project/anime-offline-database/master/anime-offline-database-minified.json) which includes AniDB IDs and cross-references. Alternatively, scrape `https://anidb.net/anime/?type.tvseries=1&orderby.rating=0.2&noalias=1` for top-rated anime and extract their OP/ED theme titles.
   - Use the `anisongdb.com` API as primary source: `GET https://anisongdb.com/api/filter?anime_search_filter={}&category_filter=Opening,Ending&sort=popularity` — this indexes AniDB-linked anime themes with song metadata.
   - Return an array of `{ title, artist, anime, anidb_rank }` sorted by popularity.
   - Rate limit appropriately (1 req/s for scraping, p-limit(1)).

5. Create `scripts/lib/youtube-search.ts`:
   - Function `searchYouTubeVideoId(title: string, artist: string)`: calls YouTube Data API v3 `GET /youtube/v3/search?q={title} {artist} official&type=video&maxResults=1&key={YOUTUBE_API_KEY}`. Returns the first video ID or null.
   - Uses `process.env.YOUTUBE_API_KEY`.

6. Create `scripts/seed/01-build-manifest.ts`:
   - Load dotenv from `.env.local`.
   - Read existing `data/songs-manifest.json` if it exists (checkpoint/resume pattern — skip songs already in the manifest).
   - **Multi-source ranking (locked decision: pull from MyAnimeList, Spotify, and AniDB):**
     a. Fetch top 50 anime by popularity from Jikan (2 pages of 25). For each anime, fetch themes. Collect all openings and endings. Assign `mal_rank` based on anime popularity position.
     b. Fetch anime themes from Spotify playlists via `fetchSpotifyAnimeThemes()`. Assign `spotify_rank` from Spotify popularity score.
     c. Fetch anime themes from AniDB via `fetchAnidbThemes()`. Assign `anidb_rank` from AniDB popularity.
     d. Merge all three lists by `title + artist` (fuzzy match — normalize unicode, strip parentheticals, case-insensitive). For each unique song, record which sources found it and at what rank in `source_rankings: { mal_rank, spotify_rank, anidb_rank }`.
     e. Compute a combined popularity score: songs found in multiple sources rank higher. Use weighted formula: `combined = (mal_rank_norm * 0.4) + (spotify_rank_norm * 0.35) + (anidb_rank_norm * 0.25)` where `_norm` is normalized to 0-1 range (lower is more popular). Songs missing from a source get a penalty score of 1.0 for that source.
     f. Sort by combined score ascending. Take top 200 unique songs.
   - Parse each Jikan theme string via `parseThemeString`. Deduplicate by `title + artist`.
   - For each song not already in the manifest: search YouTube for the video ID. Generate a slug from `anime-title-artist` in kebab-case.
   - Respect YouTube quota: 200 searches = 20,000 units. If quota is 10,000/day, the script saves progress to `data/songs-manifest.json` after every 10 songs (checkpoint saves). Log how many songs remain and estimated quota usage.
   - Validate final manifest against `SongManifestSchema`. Write to `data/songs-manifest.json`.
   - Run via: `npx tsx scripts/seed/01-build-manifest.ts`.
  </action>
