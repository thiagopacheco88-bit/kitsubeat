<objective>
Build the WhisperX timing extraction pipeline that produces word-level timestamps for every song.

Purpose: Per the locked decision, timing granularity is per-word (karaoke-style). WhisperX with CTC forced alignment produces these timestamps from YouTube audio. This plan creates the Python extraction script and a TypeScript wrapper so the main pipeline can invoke it. The timing editor (Plan 05) will correct these auto-generated timestamps.

Output: `data/timing-cache/{slug}.json` files with word-level start/end timestamps, plus a TypeScript wrapper for pipeline integration.
</objective>

2. Create `scripts/seed/04-extract-timing.py` following RESEARCH.md Pattern 4:
   - Accept CLI args: `python 04-extract-timing.py <song_slug> <youtube_id> [--output-dir data/timing-cache]`
   - Step 1: Check if output file already exists (checkpoint/resume — skip if cached).
   - Step 2: Download audio via yt-dlp: `yt-dlp --extract-audio --audio-format mp3 --audio-quality 0 -o "tmp/{song_slug}.mp3" "https://www.youtube.com/watch?v={youtube_id}"`. Create `tmp/` dir if not exists.
   - Step 3: Load WhisperX model (`large-v3`), detect device (cuda if available, else cpu), set compute_type appropriately (float16 for GPU, int8 for CPU).
   - Step 4: Transcribe with `language="ja"` and `batch_size=16`.
   - Step 5: Load alignment model for Japanese, run forced alignment for word-level timestamps.
   - Step 6: Extract words array: `{ word, start, end, score }` for each word across all segments.
   - Step 7: Flag low-confidence words: if `score < 0.6`, add `"low_confidence": true` per RESEARCH.md Pitfall 1.
   - Step 8: Write JSON to `{output_dir}/{song_slug}.json`: `{ song_slug, youtube_id, words: [...], low_confidence_count, total_words }`.
   - Step 9: Copy the downloaded mp3 to `public/audio/{song_slug}.mp3` (create `public/audio/` dir if not exists) so the timing editor waveform (Plan 05) can reference it. Then clean up the original mp3 from tmp/.
   - Print summary: total words, low-confidence count, average confidence score.

3. Also create a batch runner mode: `python 04-extract-timing.py --batch <manifest_path>` that reads songs-manifest.json and processes all songs sequentially, skipping those already in timing-cache.
