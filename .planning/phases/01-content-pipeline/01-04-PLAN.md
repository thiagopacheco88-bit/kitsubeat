<objective>
Generate complete lesson content for all 200 songs via Claude Batch API and insert into the database.

Purpose: This is the core content generation step. It takes the lyrics + pre-computed tokens from Plan 02 and produces full structured lessons (translations, grammar breakdowns, vocabulary, explanations) via a single Claude Batch API call. Then it upserts everything into Neon Postgres. The database becomes the source of truth that the Phase 2 player consumes.

Output: All 200 songs in the Neon Postgres `songs` table with complete `lesson` JSONB data including translations (EN, PT-BR, ES), vocabulary, grammar points, and verse explanations.
</objective>

  <name>Task 1: Create the Claude Batch API wrapper and lesson prompt template</name>
  <files>
    scripts/lib/batch-claude.ts
    scripts/lib/lesson-prompt.ts
    package.json
  </files>
  <action>
1. Install: `npm install @anthropic-ai/sdk`.

2. Create `scripts/lib/batch-claude.ts` following RESEARCH.md Pattern 1:
   - `buildBatchRequests(songs: SongManifestEntry[], lyricsCache: Map)`: For each song, read its lyrics cache file, build the prompt via `buildLessonPrompt`, and create a `Request` object with `custom_id: song.slug`, `model: "claude-sonnet-4-6"`, `max_tokens: 8192`, and `output_config.format` set to `{ type: "json_schema", schema: LESSON_JSON_SCHEMA }`.
   - `submitBatch(requests: Request[])`: Create batch via `client.messages.batches.create({ requests })`. Return batch ID.
   - `pollBatch(batchId: string)`: Poll every 30 seconds until `processing_status !== "in_progress"`. Log progress on each poll (request counts from status).
   - `streamResults(batchId: string)`: Async generator that yields `{ custom_id, lesson }` for each succeeded result. Log and skip failed results.
   - Uses `process.env.ANTHROPIC_API_KEY`.

3. Create `scripts/lib/lesson-prompt.ts` following RESEARCH.md Complete Lesson Prompt Template:
   - `buildLessonPrompt(song: SongManifestEntry, lyrics: string, tokens: KuromojiToken[])`: Returns the full prompt string.
   - The prompt MUST include:
     a. Song metadata (title, artist, anime, season_info, year).
     b. Pre-analyzed tokens from kuroshiro as JSON context.
     c. Raw lyrics text.
     d. All 10 instructions from the research template, including:
        - Grammar color coding: noun=blue, verb=red, adjective=green, adverb=orange, particle=grey, expression=none.
        - Translation style: natural/fluent, Portuguese = Brazilian PT-BR.
        - Verse explanations: literal_meaning first, cultural_context only if genuine nuance exists.
        - JLPT assignment algorithm: "highest JLPT level required to understand 80% of vocabulary, cross-referenced with the most complex grammar pattern" with calibration examples.
        - Culturally untranslatable terms: Claude's discretion per case.
     e. Required languages in translations record: "en", "pt-BR", "es".
  </action>
  <verify>Run `npx tsx -e "import { buildBatchRequests } from './scripts/lib/batch-claude'; console.log('Batch wrapper OK')"` to confirm compilation. Verify the prompt template includes all required instruction sections by reading the generated prompt for a sample song.</verify>
  <done>Batch API wrapper handles submit/poll/stream lifecycle. Lesson prompt includes all locked decisions: grammar colors, translation style (natural/fluent), PT-BR, literal-first explanations, JLPT auto-assignment algorithm with calibration examples.</done>

1. Create `src/lib/db/index.ts`:
   - Import `neon` from `@neondatabase/serverless` and `drizzle` from `drizzle-orm/neon-http`.
   - Create and export `db` instance using `process.env.DATABASE_URL`.

2. Create `scripts/seed/03-generate-content.ts`:
   - Load dotenv from `.env.local`.
   - Read `data/songs-manifest.json`.
   - Read all lyrics cache files from `data/lyrics-cache/`.
   - Skip songs that already have a generated lesson in `data/lessons-cache/{slug}.json` (checkpoint/resume).
   - Build batch requests for all un-cached songs.
   - If no requests needed, print "All songs already generated" and exit.
   - Submit batch, poll until complete, stream results.
   - For each result, validate against LessonSchema (Zod). If validation fails, log the error and the song slug but don't crash.
   - Write each validated lesson to `data/lessons-cache/{slug}.json`.
   - Print summary: X succeeded, Y failed validation, Z skipped (already cached).
   - Cost estimate at end: log approximate input/output tokens and cost at batch pricing ($1.50/$7.50 per MTok).
   - Run via: `npx tsx scripts/seed/03-generate-content.ts`.

3. Create `scripts/seed/05-insert-db.ts`:
   - Load dotenv from `.env.local`.
   - Read `data/songs-manifest.json` and all files from `data/lessons-cache/`.
   - Read timing cache from `data/timing-cache/` (required — Plan 03 produces these files for all 200 songs).
   - For each song in manifest:
     a. Read manifest entry for metadata.
     b. Read lesson cache for lesson JSONB.
     c. Read timing cache for timing_data JSONB + timing_youtube_id (must exist — Plan 03 guarantees timing files for all songs).
     d. Upsert into songs table using `db.insert(songs).values({...}).onConflictDoUpdate({ target: songs.slug, set: { ... } })`.
   - Set `content_schema_version = 1` on all records.
   - Print summary: X inserted, Y updated, Z skipped (no lesson data).
   - Run via: `npx tsx scripts/seed/05-insert-db.ts`.
  </action>

3. Run `npx tsx scripts/seed/05-insert-db.ts` — verify it upserts records into the songs table.
