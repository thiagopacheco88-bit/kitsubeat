  <name>Task 1: Create pipeline orchestration script with checkpoint/resume</name>
  <files>scripts/seed/run-pipeline.ts</files>
  <action>
Create `scripts/seed/run-pipeline.ts` that orchestrates the full content pipeline:

1. Define the 7 pipeline steps in order:
   - Step 1: Build manifest (`01-build-manifest.ts`) -- check: `data/songs-manifest.json` exists and has entries
   - Step 2: Fetch lyrics (`02-fetch-lyrics.ts`) -- check: `data/lyrics-cache/` has files matching manifest count
   - Step 3: Extract timing (`04-extract-timing.py --batch data/songs-manifest.json`) -- check: `data/timing-cache/` has files
   - Step 4: Backfill whisper lyrics (`04b-backfill-whisper-lyrics.ts`) -- check: no `pending_whisper` entries in lyrics-cache
   - Step 5: Generate content (`03-generate-content.ts`) -- check: `data/lessons-cache/` has files matching manifest count
   - Step 6: Push DB schema (`npx drizzle-kit push`) -- check: can connect and query songs table
   - Step 7: Insert into DB (`05-insert-db.ts`) -- check: songs table row count > 0

2. For each step:
   - Check if the step's output already exists (checkpoint detection)
   - If output exists, print "[SKIP] Step N: already complete" and move to next
   - If output missing, print "[RUN] Step N: description" and execute via `execSync` with `stdio: 'inherit'`
   - If step fails (non-zero exit), print error and stop (don't continue to dependent steps)
   - After success, print "[DONE] Step N: description"

3. At the end, print a summary table showing each step's status (skipped/completed/failed).

4. Add CLI flags:
   - `--from N` to start from step N (skip checkpoint detection for earlier steps)
   - `--dry-run` to show what would run without executing
   - `--step N` to run only step N

5. Load dotenv from `.env.local` at startup. Validate required env vars exist before running:
   - Steps 1: YOUTUBE_API_KEY, SPOTIFY_CLIENT_ID, SPOTIFY_CLIENT_SECRET
   - Steps 5: ANTHROPIC_API_KEY
   - Steps 6-7: DATABASE_URL
   Print clear error messages listing which vars are missing for the requested steps.

6. Add npm script: `"pipeline": "tsx scripts/seed/run-pipeline.ts"`
  </action>
  <verify>Run `npx tsx scripts/seed/run-pipeline.ts --dry-run` and verify it lists all 7 steps with their status (all should show as "pending" since no data exists yet).</verify>
  <done>Pipeline runner script exists with checkpoint/resume, --dry-run, --from, and --step flags. It validates env vars per step and orchestrates all 7 seed scripts in sequence.</done>
