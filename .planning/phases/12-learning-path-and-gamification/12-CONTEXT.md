# Phase 12: Learning Path & Gamification — Context

**Gathered:** 2026-04-19
**Status:** Ready for planning

<domain>
## Phase Boundary

Replace the current "browse any song, any time" model with a curated **stepped path** experience, and add an **XP / streaks / levels** system that makes every session feel like measurable forward motion. Phase scaffolds **reward-slot infrastructure** that v4.0 Phase 21 (anime scenes) will later populate.

**What this phase IS:**
- A new `/path` surface (Duolingo-style stepped map) that recommends a song order across the existing catalog
- An XP/level/streak system persisted per user, surfaced via celebration moments + a HUD on `/path` and `/profile`
- A JLPT-mastery + gap-analysis dashboard (extension of the existing Phase 11.4 `/vocabulary` dashboard) that shows which vocabulary the user has mastered per JLPT tier
- Cosmetic level-up rewards (avatar borders, profile color themes, badge collection)
- Reward-slot infrastructure: typed slots that v4.0 Phase 21 fills with anime scene / cultural-vocabulary content. **Slots are invisible in v3.0 — they only render once content exists.**

**What this phase is NOT:**
- A song-locking system. Songs are NOT gated by level. The existing freemium gates (Phase 8/10 quotas) remain the only access barrier.
- A new content type. Anime scenes / cultural vocabulary live in v4.0 Phase 21.
- A new song classification. The existing `difficulty_tier` ENUM `('basic', 'intermediate', 'advanced')` on the `songs` table (in DB since `drizzle/0000_furry_zeigeist.sql`, generated by `scripts/lib/lesson-prompt.ts`) is consumed as-is. No re-tagging, no new column.

<decisions>
## Implementation Decisions

### Path structure & visualization
- **Visualization:** Duolingo-style stepped vertical map (winding path of nodes). Each node = one song. Active / next-recommended node visually highlighted.
- **Tier grouping:** Songs grouped on the path by existing `difficulty_tier` ENUM (`basic` → `intermediate` → `advanced`). Reuse the column; do NOT introduce a new song-tier taxonomy.
- **No locks:** Every node openable from day one. Level does not gate songs. The existing freemium quotas (Phase 8 song_quota / Phase 10 advanced_drill quota) are the only access barriers.
- **Path-order pull:** Strong "next up" cue (visual highlight + arrow on the map) plus a path-order XP bonus (see XP economy below). Curation by incentive, not restriction.
- **Start-here:** New user picks from **3 starter songs** (curated `basic` tier, varied anime vibes). Selection persists; that song becomes the user's first "active" path node.
- **Path + catalog coexist:** `/path` is the primary landing surface for signed-in users. `/songs` catalog stays browsable as "free explore" mode — existing filters and search continue to work unchanged.

### Song tiers vs vocabulary tiers (taxonomy split)
- **Songs:** classified `basic | intermediate | advanced` via existing `difficulty_tier` column. This drives the path grouping.
- **Vocabulary:** classified `N5 | N4 | N3 | N2 | N1` (JLPT) on individual `vocabulary_items` rows. This drives the **JLPT mastery + gap-analysis dashboard** (see below).
- **Why split:** The catalog has very few strict-N5 songs, but `intermediate` (N4) songs are full of N5 vocabulary. JLPT is the right resolution for *words*; B/I/A is the right resolution for *songs*. They are parallel taxonomies, not a single ladder.

### JLPT mastery + gap-analysis dashboard
- **Where:** Extends the existing Phase 11.4 `/vocabulary` dashboard. New view/tab grouping mastered vocab by JLPT tier and surfacing gaps (e.g., "You've mastered 87/180 N5 words — 93 to go to be 'JLPT N5 fluent'").
- **Visibility:** Always-on from day 1. Not level-gated. Not a reward slot. It's a new tool, not a level-up unlock.
- **Phase responsibility:** Built in Phase 12 (this phase), not deferred to Phase 11.4 retrofit.

### XP economy
- **XP sources (hybrid model):**
  - **Per correct exercise answer:** small drip (e.g., 1 XP per correct answer) — keeps every interaction tactile
  - **Per session completion:** small chunk (e.g., 10–20 XP) — rewards finishing
  - **Per star earned:** large chunk (e.g., 25–100 XP per star) — biggest milestone reward
  - Specific values are planner discretion; the SHAPE is hybrid drips + bursts.
- **Daily soft cap:** Full XP up to ~250 XP/day, then 25% of normal rate beyond that. Discourages marathon-then-burnout without punishing enthusiasm. Specific cap value is planner discretion.
- **Level curve:** Gentle exponential, Duolingo-style. Each level requires ~20% more XP than the previous. Specific base + exponent are planner discretion.
- **Bonus multipliers** (all four enabled):
  - **Daily-first session bonus:** First session of the day earns 1.5× XP. Reinforces the streak loop.
  - **Perfect run bonus:** Session with 0 wrong answers earns +25% XP.
  - **Path-order bonus:** Completing the next recommended path node earns +25% XP. The lever that makes path-following feel rewarded without being mandatory.
  - **Streak-milestone bonus:** Hitting 7-day / 30-day / 100-day streaks grants discrete one-time XP awards (specific amounts: planner discretion).

### Streak mechanics
- **Threshold for a "streak day":** ONE completed exercise session (any size, any song). Just opening doesn't count; finishing does.
- **Day rollover:** Auto-detected device timezone (`Intl.DateTimeFormat().resolvedOptions().timeZone`). Streak day rolls at the user's local midnight.
- **Grace day (per ROADMAP success criterion 3):** ONE per week, **auto-applied silently** when missed. User does not need to "use" the grace token — system applies it on their behalf if a day is missed within the week. Next session shows a brief "phew, your streak survived" acknowledgment.
- **Streak break (no grace left):** Silent reset to 0. Streak widget always shows the user's high-water-mark ("Best: 47 days") as a permanent record. No modal, no upsell, no shame.
- **Schema implication for planner:** Will need at minimum a `streak_current`, `streak_best`, `last_streak_day_local` (date in user TZ), and `grace_used_this_week` field — planner to confirm exact shape.

### Level-up celebration & unlocks
- **What unlocks per level in v3.0:** **Cosmetic-only.** Avatar borders, profile color themes, badge collection. No songs, no functional features, no v4.0-dependent content.
- **Reward-slot scaffolding:** Build the typed-slot infrastructure (slot type, level threshold per slot, slot content interface). Slots that have no content in v3.0 **do not render at all** — no "coming soon" placeholders, no empty cards. v4.0 Phase 21 ships the content that makes slots appear.
- **Unlock trigger:** Level-based. Each level threshold may unlock 0 or more slots. The mapping is data-driven so v4.0 can wire scenes/cultural content to specific levels without code changes.
- **Celebration tone:** **Always full-screen takeover** on every level-up. No tier-only treatment. Every level-up gets the same theatrical moment.
- **Audio + haptics:** Sound effect + visual flash + (mobile) haptic vibration. Defaults ON but **configurable** in profile settings (mute sound, disable haptics independently).
- **Reuse:** The existing Phase 10 confetti / star-shine primitive should be reused or extended where animation overlap exists; do not introduce a second confetti library. If a new sound/haptic API is needed, it's net-new — but visual primitives reuse first.

### HUD (where XP / level / streak surface during normal play)
- **HUD lives on `/path` and `/profile` ONLY.** Header bar showing level number, XP progress to next level, and streak flame icon.
- **NOT on `/songs` catalog** — keeps the existing browse experience uncluttered.
- **NOT on the song player** (`/songs/[slug]`) — keeps player chrome focused on the lesson and exercises. The end-of-session summary already shows XP gained.
- **End-of-session summary:** Continues to show XP gained, current streak, level progress, and a preview of the next reward slot (if one is upcoming) — per ROADMAP success criterion 2.

### Claude's Discretion
The planner / researcher should make calls on:
- Specific XP values (per answer, per session, per star, daily cap, streak-milestone amounts)
- Exact level curve coefficient (e.g., `xp_for_level_n = 100 * 1.2^n`)
- Schema column names for streak / XP / level / cosmetic-unlock state
- Which 3 songs become the "starter pick" (subject to user review before commit)
- Cosmetic content catalog (avatar borders, color themes, badges) — planner proposes, user reviews
- Exact reward-slot data shape (must support v4.0 Phase 21 anime-scene content as a non-breaking extension)
- Whether to reuse the existing PostHog/event pipeline (Phase 15) or temporarily emit XP events to a stub until Phase 15 lands

</decisions>

<specifics>
## Specific Ideas

- **Reference: Duolingo path** — the stepped vertical map metaphor is the visual reference. Don't copy its tone, but copy its "next node highlighted, others visible" interaction model.
- **Cultural-content vision** — the user's original framing of unlocks was "anime references like character-name etymology (Kisame = 'demon shark', Master Kame = 'turtle'), Japanese moves explained in Japanese, Japanese mythology references." This is **v4.0 Phase 21 content**, not v3.0 work. The reward-slot infrastructure built here must support that content shape when it arrives.
- **JLPT gap framing** — the user wants the dashboard to surface gaps in a goal-oriented way: "You've mastered X words to be considered N5 fluent — Y words remaining." Frame as progress toward a JLPT competency standard, not just a counter.
- **Cosmetic unlocks should feel earned, not generic** — avoid "level 3: silver border, level 4: gold border" pure-tier scaling. Prefer themed cosmetics tied to anime aesthetics where possible (still authoring within v3.0 — no licensed art needed; abstract anime-inspired patterns are fine).

</specifics>

<deferred>
## Deferred Ideas

- **Anime cultural content** (character-name etymology, Japanese moves explained, mythology references) — v4.0 Phase 21 (Anime Scenes & Cultural Vocabulary). The reward-slot infrastructure built in Phase 12 hosts this content when it arrives.
- **Functional unlocks via levels** (e.g., L5 unlocks JLPT gap dashboard, L10 unlocks dictionary customization) — explicitly rejected in this discussion. JLPT gap dashboard is always-on; other "functional unlock" ideas can be revisited as new phases.
- **Branching path / multiple parallel tracks** (e.g., shonen branch vs slice-of-life branch) — single linear path for v1; branching is its own future phase.
- **Streak repair as a premium feature** — explicitly rejected; silent reset preserves the "no shame" principle.
- **Diagnostic onboarding test** (placement quiz) — fixed 3-starter-pick chosen instead. Diagnostic is a future phase if signal warrants.

</deferred>
