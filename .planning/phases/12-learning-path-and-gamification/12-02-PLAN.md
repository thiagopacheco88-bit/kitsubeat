  <name>Feature 1: XP calculator + daily soft cap</name>
  <files>
    src/lib/gamification/xp.ts
    src/lib/gamification/__tests__/xp.test.ts
  </files>
  <behavior>
    `calculateXp(input)` where `input: { correctAnswers:number, sessionType:'short'|'full', newStars:0|1|2|3, previousStars:0|1|2|3, multipliers:{ dailyFirst:boolean, perfectRun:boolean, pathOrder:boolean }, xpTodayBefore:number }` returns `{ xpBase:number, xpAfterMultipliers:number, xpAfterCap:number, cappedAt:number | null }`.

    Constants (from RESEARCH Section "Recommendations 1"):
      PER_ANSWER = 2
      SESSION_SHORT = 15
      SESSION_FULL = 25
      STAR_BONUS = { 1: 30, 2: 60, 3: 100 } // only NEW stars (newStars - previousStars, clamped ≥0)
      DAILY_SOFT_CAP = 250
      ABOVE_CAP_RATE = 0.25
      STREAK_MILESTONE_BONUS = { 7: 50, 30: 150, 100: 400 } // applied separately in streak.ts

    Rules:
      - xpBase = correctAnswers * PER_ANSWER + (sessionType === 'full' ? SESSION_FULL : SESSION_SHORT) + sum(STAR_BONUS[s] for s in newStars that weren't previousStars)
      - Multiplier stack (deterministic order): xpAfterMultipliers = xpBase * (dailyFirst?1.5:1) * (perfectRun?1.25:1) * (pathOrder?1.25:1), rounded floor
      - Soft cap: if xpTodayBefore < DAILY_CAP, fill to cap at 100% then add overflow at 25%. xpAfterCap = min(DAILY_CAP - xpTodayBefore, xpAfterMultipliers) + max(0, xpAfterMultipliers - (DAILY_CAP - xpTodayBefore)) * 0.25, floored. If xpTodayBefore ≥ DAILY_CAP, entire amount at 25%. Set `cappedAt = xpTodayBefore < DAILY_CAP && xpAfterMultipliers > (DAILY_CAP - xpTodayBefore) ? DAILY_CAP : (xpTodayBefore >= DAILY_CAP ? xpTodayBefore : null)`.

    Cases (RED test file):
      - Session with 10 correct answers, short, 0 new stars, no multipliers, xpTodayBefore=0 → xpBase = 10*2+15 = 35; xpAfterMultipliers = 35; xpAfterCap = 35; cappedAt = null
      - Session with 20 correct, full mode, earns star 1 (prev 0), perfectRun=true, xpTodayBefore=0 → xpBase = 20*2+25+30 = 95; xpAfterMultipliers = 95*1.25 = 118 (floor); xpAfterCap = 118; cappedAt = null
      - Session that would push xpToday from 200 to 300 → first 50 XP at 100% + remaining 50 XP at 25% = 50 + 12 (floor) = 62; cappedAt = 250
      - Session at xpToday=300 already above cap, earns 80 → 80 * 0.25 = 20; cappedAt = 300
      - newStars < previousStars (regression, shouldn't happen but defensive) → star bonus = 0, never negative
      - All 3 multipliers enabled, 10 correct, short, no stars → 35 * 1.5 * 1.25 * 1.25 = 82 (floor)
  </behavior>
  <implementation>
    Pure function + exported constants. No imports from src/lib/db. Uses `Math.floor`. Commit cycle: RED test → GREEN impl → REFACTOR (extract `applyDailyCap` helper if test readability improves).
  </implementation>
</feature>

    `xpForLevel(n:number):number` returns the XP required to REACH level n (starting from level n-1). Formula: `Math.floor(100 * Math.pow(1.2, n-1))`.
    `levelFromXp(total:number):number` returns the highest level reachable with cumulative XP. Level 1 requires 0 total XP. Reaching level 2 requires `xpForLevel(2) = 120` additional XP on top of the 100 for level 2's own threshold — inclusive cumulative model:
      level 1 requires 0 total XP (floor)
      level 2 requires 100 total XP
      level 3 requires 100 + 120 = 220 total XP
      level 4 requires 100 + 120 + 144 = 364 total XP
      etc.
    `xpWithinCurrentLevel(total:number):{ currentLevel:number, xpInLevel:number, xpToNext:number }` returns the progress within the current level for the HUD progress bar.

    Cases:
      - levelFromXp(0) === 1
      - levelFromXp(99) === 1
      - levelFromXp(100) === 2
      - levelFromXp(219) === 2
      - levelFromXp(220) === 3
      - xpWithinCurrentLevel(150) === { currentLevel: 2, xpInLevel: 50, xpToNext: 120 }
      - xpForLevel(1) === 100; xpForLevel(2) === 120; xpForLevel(10) === 516 (floor of 100*1.2^9 ≈ 516.1)

  <name>Feature 3: Streak state machine</name>
  <files>
    src/lib/gamification/streak.ts
    src/lib/gamification/__tests__/streak.test.ts
  </files>
  <behavior>
    `advanceStreak(state, input)` where:
      state = { streakCurrent:number, streakBest:number, lastStreakDate:string|null, streakTz:string|null, graceUsedThisWeek:boolean, streakWeekStart:string|null }
      input = { tz:string, now:Date } (test injects Date for determinism)
    returns { newState:<state>, graceApplied:boolean, milestoneHit: 7|30|100|null }

    Compute `today = localDateFromTz(tz, now)` as 'YYYY-MM-DD' using Intl.DateTimeFormat('en-CA', { timeZone: tz, dateStyle: 'short' }) split-and-reorder per RESEARCH E2.
    Compute `todayWeekStart = isoWeekStart(today)` — Monday date of ISO week.

    Rules (in order):
      1. If streakWeekStart !== todayWeekStart → reset graceUsedThisWeek=false, set streakWeekStart=todayWeekStart (new week).
      2. If lastStreakDate === null (first session ever) → newState.streakCurrent = 1, lastStreakDate = today; milestone check.
      3. If lastStreakDate === today → no change (same-day).
      4. If lastStreakDate === today - 1 (direct next-day) → increment streakCurrent, set lastStreakDate=today; milestone check.
      5. If lastStreakDate === today - 2 AND graceUsedThisWeek=false → grace applied: streakCurrent stays same, lastStreakDate=today, graceUsedThisWeek=true, graceApplied=true; milestone check.
      6. Otherwise (gap ≥ 2 OR 1-day gap with grace used) → silent reset: streakCurrent = 1, lastStreakDate = today, graceApplied=false.

    Always set: streakBest = max(streakBest, streakCurrent). streakTz = tz.
    Milestone hit: one of [7, 30, 100] if streakCurrent crosses that value (exact match) — signals Plan 04 to award bonus XP.

    Helpers exported:
      - `localDateFromTz(tz:string, d:Date):string` → 'YYYY-MM-DD'
      - `isoWeekStart(dateStr:string):string` → Monday 'YYYY-MM-DD' using Intl

    Cases (TZ 'America/Sao_Paulo' pinned for tests):
      - First-ever session → streakCurrent 0→1, lastStreakDate set, milestoneHit null
      - Same day second session → streakCurrent unchanged, lastStreakDate unchanged
      - Next day → streakCurrent 1→2
      - 2-day gap, grace available → grace applied, streakCurrent unchanged, graceUsedThisWeek=true
      - 2-day gap, grace used → silent reset to 1
      - 3-day gap → silent reset regardless
      - New ISO week → graceUsedThisWeek resets to false on first session of the new week
      - streak hits 7 for the first time → milestoneHit=7; already at 7, same-day → milestoneHit=null
      - streakBest never decreases even on reset (was 10, current resets to 1, streakBest remains 10)
      - DST boundary: 23:30 local pre-DST, next session 23:30 local post-DST same calendar day → treated as same day
      - TZ jump (Tokyo → London) within same calendar day → no streak break (same 'YYYY-MM-DD')
  </behavior>
  <implementation>
    Pure function. Date arithmetic via UTC-anchored Date objects adjusted to the user's TZ via Intl.DateTimeFormat — no `date-fns-tz` dep (confirmed in RESEARCH E2). ISO week Monday via Intl weekday lookup + day-offset.
  </implementation>
</feature>
