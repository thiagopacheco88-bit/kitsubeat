  <action>
Open `scripts/types/lesson.ts` and locate the existing `VocabEntrySchema` (the Zod schema used by the lesson-generation pipeline to validate per-song vocabulary entries). Add two optional fields that mirror the shapes defined in `scripts/types/enrich.ts` (from plan 02) — but inline here, because `scripts/types/lesson.ts` must not import from another scripts module at runtime in ways that change the existing imports.

Add at the top of the file (near existing schemas) — only if not already imported — local schemas matching the enrichment shape:

```typescript
const EnrichLocalizableSchema = z.object({
  en: z.string().min(1),
  "pt-BR": z.string().min(1),
  es: z.string().min(1),
});

const EnrichKanjiCharSchema = z.object({
  char: z.string().refine((c) => [...c].length === 1),
  meaning: EnrichLocalizableSchema,
  on_yomi: z.string(),
  kun_yomi: z.string(),
  jlpt_level: z.enum(["N5", "N4", "N3", "N2", "N1"]).nullable(),
  radical_hint: EnrichLocalizableSchema,
});

const EnrichKanjiBreakdownSchema = z.object({
  characters: z.array(EnrichKanjiCharSchema).min(1),
  compound_note: EnrichLocalizableSchema.optional(),
});
```

Then extend `VocabEntrySchema` by adding two optional fields (do NOT remove or reorder existing ones):

```typescript
export const VocabEntrySchema = z.object({
  // ...existing fields (surface, reading, romaji, part_of_speech, jlpt_level, meaning, example_from_song, additional_examples, ...)...
  mnemonic: EnrichLocalizableSchema.optional(),
  kanji_breakdown: EnrichKanjiBreakdownSchema.nullable().optional(),
});
```

Why `.optional()` on BOTH fields here even though plan 02's `VocabEnrichmentSchema` requires mnemonic: generation-time validation is a superset. An LLM response that omits these fields still passes validation so we don't block existing song generation; the Wave 4 QA gate catches coverage gaps. A response that DOES include them gets shape-validated strictly.

Rebuild the JSON schema export if this file exports a `LESSON_JSON_SCHEMA` — the `zodToJsonSchema` call at the bottom will automatically pick up the new optional fields.
