## Accordion DOM Snapshot (What Users See)

When a user clicks "More" after answering a question for an enriched vocab word (e.g. 鉄道 = "railway"):

```
[feedback panel — inline]
  ✓ Correct!
  [vocab block with TierText + MasteryDetailPopover]
  鉄道 means railway.
  [Continue]  [Hide details]        ← accordion open
  ┌─────────────────────────────────────┐
  │ MEMORY TIP                          │
  │ "Iron path connecting cities..."    │
  ├─────────────────────────────────────┤
  │ KANJI BREAKDOWN                     │
  │ 鉄  てつ / -    iron    iron+metal  │
  │ 道  どう / みち  road   movement    │
  │ Compound: path made of iron tracks  │
  └─────────────────────────────────────┘
```

For kana-only words (e.g. アニメ): only Memory tip section shown, kanji breakdown absent entirely.
For unenriched vocab: "More" button hidden — no accordion rendered.

## SQL Query (Sample)

```sql
SELECT id, mnemonic, kanji_breakdown
FROM vocabulary_items
WHERE id IN (
  'uuid-1', 'uuid-2', 'uuid-3', ...
)
```

Issued once per song page load. Number of IDs = unique vocab_item_ids across all versions of the song (typically 15-25 per TV version).
