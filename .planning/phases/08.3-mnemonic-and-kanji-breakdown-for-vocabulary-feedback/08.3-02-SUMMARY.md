# Phase 08.3 Plan 02: Vocab Enrichment Script Summary

**One-liner:** Idempotent enrichment script calling inline Claude haiku API to populate mnemonic + kanji_breakdown for all vocabulary_items rows.

## What Was Built

### Task 1 — `scripts/types/enrich.ts`

Zod validation contract for LLM enrichment output:

- `LocalizableSchema` — requires `en`, `pt-BR`, `es` all non-empty (inline, no cross-boundary import)
- `KanjiCharEntrySchema` — `char` validated as exactly one Unicode code point using `[...c].length === 1` (spread iterator handles multi-byte CJK correctly)
- `KanjiBreakdownSchema` — `characters: min(1)`, `compound_note: optional()` (single-kanji words don't need it)
- `VocabEnrichmentSchema` — `mnemonic: LocalizableSchema`, `kanji_breakdown: KanjiBreakdownSchema.nullable()`
- `ENRICH_JSON_SCHEMA` derived via `zodToJsonSchema` with `$refStrategy: "none"` — suitable for Anthropic structured output config

Verified: `tsc --noEmit` on scripts tsconfig shows zero errors for enrich.ts; `tsx` loads the module cleanly (exit 0).

### Task 2 — `scripts/seed/11-enrich-vocab.ts` + `package.json`

Idempotent per-row enrichment script:

- **Skip gate:** `isNull(vocabularyItems.mnemonic)` — the sole skip signal. Kana-only words legitimately keep `kanji_breakdown = null` forever, so that column cannot be the gate.
- **Inline API:** `client.messages.create()` only. Zero references to `client.messages.batches`.
- **Per-row commit:** `db.update().where(eq(vocabularyItems.id, row.id))` executes and awaits before the next row starts. Restart picks up remaining NULL-mnemonic rows automatically.
- **Kanji detection:** `KANJI_RE = /[\u4e00-\u9fff]/` — same regex used by `backfill-vocab-identity.ts`.
- **Concurrency:** `pLimit(5)` to avoid Anthropic 429s on ~705 rows.
- **Retry:** `maxRetries: 3` on the Anthropic client handles transient 429s.
- **API key guard:** exits immediately with instructions if `ANTHROPIC_API_KEY` is not set.

`package.json` script registered: `"seed:enrich-vocab": "tsx --tsconfig tsconfig.scripts.json scripts/seed/11-enrich-vocab.ts"` — matches `backfill:vocab` / `audit:geo` invocation style.

## Authentication / Operator Gates

**ANTHROPIC_API_KEY required to run enrichment against production DB.**

The script ships fully ready but cannot call the API without the key. This is documented as a blocker in STATE.md.

To run enrichment:
1. Ensure `ANTHROPIC_API_KEY` is set in `.env.local` (or exported in shell)
2. Ensure `DATABASE_URL` / `POSTGRES_URL` points to the target DB
3. `npm run seed:enrich-vocab`
4. Verify: `psql $DATABASE_URL -c "SELECT COUNT(*) FROM vocabulary_items WHERE mnemonic IS NULL"` → 0
5. Second run should log `0 rows pending enrichment` (idempotency proof)

Expected: first run ~705 rows, ~10-15 min at concurrency=5.
