   - Index on `due` named `user_vocab_mastery_due_idx`
   - Index on `(user_id, due)` named `user_vocab_mastery_user_due_idx`

5. Add the `userExerciseLog` table:
   - `id`: uuid PK defaultRandom

   - `id`: uuid PK defaultRandom
   - `user_id`: text notNull unique
   - `plan`: text notNull â€” e.g., "free", "premium_monthly", "premium_annual"
   - `status`: text notNull â€” e.g., "active", "canceled", "past_due", "trialing"
   - `provider`: text â€” e.g., "stripe", "lemon_squeezy" (nullable for free)
   - `provider_subscription_id`: text (nullable)
   - `provider_customer_id`: text (nullable)
   - `current_period_start`: timestamp with timezone
   - `current_period_end`: timestamp with timezone
   - `created_at` and `updated_at`: timestamp with timezone, defaultNow, notNull

7. Add the `vocabGlobal` materialized view using `pgMaterializedView`:
   - Columns: vocab_item_id (uuid), song_id (uuid), version_type (text), dictionary_form (text), reading (text), jlpt_level (text)
   - SQL body joins song_versions with vocabulary_items via LATERAL jsonb_array_elements on the lesson vocabulary array, matching on vocab_item_id field in the JSONB (after backfill patches it in)
   - Use this SQL for the view definition:
     ```sql
     SELECT
       vi.id AS vocab_item_id,
       sv.song_id,
       sv.version_type,
       vi.dictionary_form,
       vi.reading,
       vi.jlpt_level
     FROM song_versions sv
     CROSS JOIN LATERAL jsonb_array_elements(sv.lesson->'vocabulary') AS elem
     JOIN vocabulary_items vi ON vi.id = (elem->>'vocab_item_id')::uuid
     WHERE sv.lesson IS NOT NULL
       AND elem->>'vocab_item_id' IS NOT NULL
     ```

8. Export all new tables and types: `VocabularyItem`, `NewVocabularyItem`, `UserVocabMastery`, `NewUserVocabMastery`, `UserExerciseLog`, `NewUserExerciseLog`, `Subscription`, `NewSubscription`.
