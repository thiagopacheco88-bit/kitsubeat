<task type="auto">
  <name>Task 1: Create conjugation path parser</name>
  <files>scripts/lib/conjugation-audit.ts</files>
  <action>
Create `scripts/lib/conjugation-audit.ts` with:

1. Export the `StructuredConjugation` interface:
   ```typescript
   export interface StructuredConjugation {
     base: string;           // dictionary form (e.g., 食べる)
     conjugated: string;     // conjugated form (e.g., 食べた)
     conjugation_type: string; // type label (e.g., "past tense", "te-form → te-iru")
     is_structured: boolean;   // true if successfully parsed
   }
   ```

2. Export `parseConjugationPath(path: string | undefined): StructuredConjugation | null`:
   - If `path` is undefined/null/empty string: return null
   - If path contains `→` (arrow character):
     - Split on `→`, trim each part
     - If 2+ parts and first part looks like a Japanese word (contains CJK/hiragana/katakana):
       - `base` = first part
       - `conjugated` = last part
       - `conjugation_type` = all parts after the first joined with " → "
       - `is_structured = true`
     - If first part is a grammar pattern name (no Japanese characters, e.g., "dictionary form → te-form"):
       - `is_structured = false`, `conjugation_type = "unstructured"`, both base and conjugated = path
   - If no `→`:
     - Return `{ base: path, conjugated: path, conjugation_type: "unstructured", is_structured: false }`

3. Export `auditConjugationPaths(grammarPoints: Array<{ name: string; conjugation_path?: string }>)`:
   - Returns `{ parsed: StructuredConjugation[], skipped: string[], total: number }`
   - Iterates grammar points, calls parseConjugationPath on each
   - Collects parsed results and skipped entries (the raw path strings that were unstructured)

4. Use a helper regex to detect Japanese characters: `/[\u3040-\u309F\u30A0-\u30FF\u4E00-\u9FFF]/` (hiragana + katakana + CJK unified)
  </action>
  <verify>
Run a quick inline test:

<task type="auto">
  <name>Task 2: Create idempotent vocabulary backfill script</name>
  <files>scripts/backfill-vocab-identity.ts</files>
  <action>
Create `scripts/backfill-vocab-identity.ts` following the established pattern from `scripts/seed/05-insert-db.ts`:

1. **Setup:**
   - `import { config } from "dotenv"; config({ path: ".env.local" });`
   - Import `getDb` from `src/lib/db/index.js`
   - Import schema tables: `vocabularyItems`, `songVersions`, `vocabGlobal` from `src/lib/db/schema.js`
   - Import `LessonSchema` from `scripts/types/lesson.js`
   - Import `parseConjugationPath`, `auditConjugationPaths` from `scripts/lib/conjugation-audit.js`
   - Import `sql`, `eq`, `and` from `drizzle-orm`
   - Parse `--dry-run` flag from process.argv

2. **Main function flow:**

   a. **Load all song versions with lessons:**
      ```typescript
      const db = getDb();
      const rows = await db.select().from(songVersions).where(sql`lesson IS NOT NULL`);
      ```

   b. **Phase 1: Extract and deduplicate vocabulary → vocabulary_items table**
      - For each row, parse lesson with `LessonSchema.safeParse(row.lesson)`
      - Skip rows that fail validation (count as `skippedRows`)
      - For each `VocabEntry` in `lesson.vocabulary`:
        - Determine `dictionary_form`: use `entry.surface` (this is the surface form from the lesson — per user decision, identity is by dictionary form. The surface in VocabEntry IS the dictionary form since vocabulary entries list the base form, not conjugated forms)
        - Detect katakana loanwords: if `dictionary_form` matches `/^[\u30A0-\u30FF\u30FC]+$/` (all katakana + prolonged sound mark), set `is_katakana_loanword = true`
        - Build a dedup key: `${dictionary_form}|${entry.reading}`
        - Collect unique entries in a Map to avoid duplicate inserts within the same batch
      - If not `--dry-run`: batch insert all unique entries using `db.insert(vocabularyItems).values(batch).onConflictDoNothing()` in chunks of 50
      - For entries that already exist (onConflictDoNothing returns no id), resolve their UUIDs with a SELECT query

   c. **Phase 2: Patch JSONB with vocab_item_id**
      - Build a lookup Map: `Map<string, string>` where key = `${dictionary_form}|${reading}` → value = UUID
      - For each song version row:
        - Re-parse the lesson
        - For each vocab entry, look up the UUID and set `entry.vocab_item_id = uuid`
        - If not `--dry-run`: update the song_versions row with the patched lesson JSONB:
          ```typescript
          await db.update(songVersions)
            .set({ lesson: patchedLesson, updated_at: new Date() })
            .where(eq(songVersions.id, row.id));
          ```
      - Process in batches of 10-20 rows to avoid Neon connection timeout

   d. **Phase 3: Audit grammar conjugation paths**
      - Collect all grammar_points from all parsed lessons
      - Call `auditConjugationPaths(allGrammarPoints)`
      - Store structured conjugation data in the JSONB — add a `structured_conjugation` field to each grammar point that has a parseable path:
        - Actually, per user decision, structured data is just `{ base, conjugated, conjugation_type }` — store this in a field on the grammar point in JSONB. BUT: the GrammarPoint type doesn't have this field yet.
        - DECISION: Do NOT mutate GrammarPoint in JSONB. Instead, just produce the console summary. The conjugation audit data lives in the parser output and will be consumed at exercise generation time (Phase 10) by calling `parseConjugationPath` on the existing `conjugation_path` field. This avoids another type mutation.

   e. **Phase 4: Refresh materialized view**
      - If not `--dry-run`:
        ```typescript
        await db.refreshMaterializedView(vocabGlobal);
        ```
      - Note: Use non-concurrent refresh on first run (view may be empty). After first population, concurrent refresh can be used.

3. **Console summary** (always printed, even in dry-run):
   ```
   === Vocabulary Backfill Summary ===
   Song versions processed: X
   Song versions skipped (invalid lesson): Y
   Unique vocabulary items: Z (N new, M existing)
   Katakana loanwords flagged: K
   JSONB rows patched with vocab_item_id: P
   
   === Conjugation Audit Summary ===
   Total grammar points: T
   Parseable (structured): S (XX%)
   Skipped (unstructured): U (YY%)
   
   Examples of skipped entries:
     - "〜ている (te-iru form)"
     - "dictionary form → te-form"
     [up to 5 examples]
   
   Materialized view refreshed: yes/no (dry-run)
   ```

4. **Add npm script** in package.json:
   ```json
   "backfill:vocab": "tsx scripts/backfill-vocab-identity.ts"
   ```

5. **Idempotency guarantees:**
   - `onConflictDoNothing` for vocabulary_items inserts — safe to re-run
   - JSONB patching overwrites vocab_item_id — same UUID re-applied is a no-op
   - Materialized view refresh is always safe to re-run
