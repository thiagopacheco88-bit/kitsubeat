# Phase 08.2 Plan 03: FSRS UI Wiring — Tier Rendering, Reveal Hatch, Mastery Popovers Summary

Progressive-disclosure UX shipped: TierText renders vocab at FSRS-derived tier, ExerciseTab prefetches tiers in one call, QuestionCard fires recordVocabAnswer per answer with optimistic tier update, reveal hatch downgrades to Again, FeedbackPanel and SessionSummary expose tap-to-inspect mastery popovers.

## What Was Built

### 1. `src/app/songs/[slug]/components/TierText.tsx` — Tier-Aware Vocab Renderer

**Props:**
```ts
interface TierTextProps {
  vocab: VocabInfo;          // { surface, reading, romaji, vocab_item_id? }
  tier: Tier;                // 1|2|3 from @/lib/fsrs/tier
  mode: "prompt" | "option" | "feedback";
  forceTier1?: boolean;      // beats stored tier — feedback panel invariant
  forceKanjiOnly?: boolean;  // beats stored tier — reading_match + meaning→vocab options
}
```

**Render rules:**
| effectiveTier | Surface | Furigana | Romaji |
|---|---|---|---|
| 1 | yes | yes (unless pure-kana) | yes |
| 2 | yes | yes (unless pure-kana) | no |
| 3 | yes | no | no |

**Leak-override matrix (locked per CONTEXT):**
| Exercise type | Prompt | Options |
|---|---|---|
| vocab_meaning | tier (or effectiveTier if revealed) | — (string options) |
| reading_match | forceKanjiOnly (furigana would reveal romaji answer) | — (romaji strings) |
| meaning_vocab | — (meaning string, raw) | forceKanjiOnly |
| fill_lyric | — (verse+blank, raw) | forceKanjiOnly |
| FeedbackPanel vocab block | forceTier1 | — |

**Pure-kana edge case:** When `vocab.surface === vocab.reading`, furigana is omitted at every tier. Romaji still shows at Tier 1.

### 2. `src/app/songs/[slug]/components/MasteryDetailPopover.tsx` — Tap-to-Inspect

**Props:** `{ vocabItemId: string; userId: string; trigger: ReactNode }`

**Behavior:**
- Renders trigger in a button (underline decoration). On click, toggles panel.
- On open, fetches `/api/exercises/vocab-mastery/{id}?userId={userId}`.
- Shows: State (New/Learning/Review/Relearning), Reps, Correct%, Next due (relative).
- Loading skeleton while fetching. "Mastery details unavailable" on error.
- Closes on outside click or Escape.

**Visual:** `absolute z-20 w-64 rounded-lg border border-gray-700 bg-gray-900 p-3 text-xs` — matches dark exercise palette.

### 3. Session Store — Tier Slice (`src/stores/exerciseSession.ts`)

New state:
```ts
tiers: Record<string, Tier>;            // keyed by vocabItemId
revealedQuestionIds: Record<string, true>; // keyed by question.id
```

New actions:
```ts
setTiers(tiers: Record<string, Tier>): void;
setTier(vocabItemId: string, tier: Tier): void;
markRevealed(questionId: string): void;
```

`startSession` resets both slices (new session starts clean). `clearSession` resets both.
Both slices persist (so a paused session resumes at correct tier without re-fetch).

### 4. Generator — VocabInfo type + distractorVocab map (`src/lib/exercises/generator.ts`)

Added `VocabInfo` interface (`surface, reading, romaji, vocab_item_id?`) exported from generator.

Added to `Question`:
- `vocabInfo: VocabInfo` — target word for TierText rendering
- `distractorVocab?: Record<string, VocabInfo>` — distractor surface → VocabInfo for option rendering and FeedbackPanel wrong-pick popover

`pickDistractorsWithVocab` internal helper preserves the existing `pickDistractors` public API while also threading VocabInfo through to `makeQuestion`.

### 5. ExerciseTab — Tier Prefetch

In `handleStart`, after `buildQuestions` and after `startSession`:
1. Collect unique vocabItemIds from questions
2. Fetch `/api/exercises/vocab-tiers?ids=...&userId=...`
3. Call `store.setTiers(tiers)` — placed AFTER `startSession` so the reset doesn't wipe tiers
4. On failure: log + continue — every word defaults to Tier 1 (cold-start)

### 6. QuestionCard — Tier Rendering, Reveal Hatch, Inline FSRS Commit

- Pulls `tiers`, `revealedQuestionIds`, `markRevealed`, `setTier` from store
- Computes `effectiveTier = revealed ? 1 : targetTier`
- Renders prompts through TierText (or raw for meaning_vocab meaning / fill_lyric verse)
- Renders options through TierText with forceKanjiOnly for meaning_vocab / fill_lyric
- "Reveal reading" button visible when: unanswered AND targetTier !== 1 AND not yet revealed
- In `handleSelect`: fires `recordVocabAnswer` non-blocking after `onAnswered`; on success calls `setTier` for optimistic mid-session update

### 7. FeedbackPanel — Vocab Block + Mastery Popover

- Vocab block: `<MasteryDetailPopover trigger={<TierText forceTier1 />} />` — always Tier 1
- Wrong-pick callout: if `distractorVocab[chosenAnswer]?.vocab_item_id` present, wraps distractor surface in `MasteryDetailPopover`
- Falls back to plain text if `vocab_item_id` is missing (legacy questions)

### 8. SessionSummary — Words Reviewed List

- Collapsible "Words reviewed (N)" disclosure after stats row
- Deduplicates by `vocabItemId` (each word shown once regardless of how many question types it appeared in)
- Each word wrapped in `MasteryDetailPopover` trigger
- Skips rows where `vocabItemId` is falsy (defensive)
