# Phase 08.2 Plan 01: FSRS Core — Rating Policy, Scheduler, and Tier Mapper Summary

Pure FSRS core: deterministic rating policy (RATING_WEIGHTS), ts-fsrs scheduler wrapper, and state→tier mapper — zero DB or React coupling, 29 vitest tests green.

## What Was Built

Three pure TypeScript modules under `src/lib/fsrs/`:

### 1. `src/lib/fsrs/rating.ts` — Rating Policy

**Public API:**
```ts
export type FSRSRating = 1 | 2 | 3 | 4;
export { Rating };  // ts-fsrs Rating enum re-exported
export const RATING_WEIGHTS: Record<ExerciseType, FSRSRating>;
export function ratingFor(
  exerciseType: ExerciseType,
  correct: boolean,
  opts?: { revealedReading?: boolean }
): FSRSRating;
```

**RATING_WEIGHTS table (locked in 08.2-CONTEXT.md):**

| ExerciseType    | Correct Rating | Label  | Rationale                          |
|-----------------|---------------|--------|-------------------------------------|
| `meaning_vocab` | 4             | Easy   | Production-flavored, hardest direction |
| `vocab_meaning` | 3             | Good   | Recognition — standard reward       |
| `fill_lyric`    | 3             | Good   | Recognition + lyric context         |
| `reading_match` | 2             | Hard   | Pure surface, easiest, lowest reward |

**Rules:**
- `revealedReading: true` → always returns 1 (Again), regardless of `correct`
- `correct === false` → returns 1 (Again)
- `correct === true` → returns `RATING_WEIGHTS[exerciseType]`

### 2. `src/lib/fsrs/tier.ts` — State→Tier Mapper

**Public API:**
```ts
export type Tier = 1 | 2 | 3;
export const TIER_NEW: Tier = 1;
export const TIER_LEARNING: Tier = 2;
export const TIER_REVIEW: Tier = 3;
export function tierFor(state: number): Tier;
```

**State→Tier mapping (locked in 08.2-CONTEXT.md):**

| FSRS State | Value | Tier | Label    |
|------------|-------|------|----------|
| New        | 0     | 1    | Full assistance (kanji + furigana + romaji) |
| Learning   | 1     | 2    | Partial (kanji + furigana)                 |
| Review     | 2     | 3    | Bare kanji only                            |
| Relearning | 3     | 2    | Collapses to Learning (lapsed card)        |
| Unknown    | other | 1    | Defensive fallback to New                  |

### 3. `src/lib/fsrs/scheduler.ts` — ts-fsrs Wrapper

**Public API:**
```ts
export interface ScheduledUpdate {
  stability: number;
  difficulty: number;
  elapsed_days: number;
  scheduled_days: number;
  reps: number;
  lapses: number;
  state: 0 | 1 | 2 | 3;
  due: Date;
  last_review: Date;
}

export interface MasteryRow {
  stability: number | null;
  difficulty: number | null;
  elapsed_days: number | null;
  scheduled_days: number | null;
  reps: number;
  lapses: number;
  state: 0 | 1 | 2 | 3;
  due: Date | null;
  last_review: Date | null;
  intensity_preset?: IntensityPreset;  // carried; ignored by scheduler
}

export function scheduleReview(
  prev: MasteryRow | null,
  rating: FSRSRating,
  intensity: IntensityPreset = "normal",
  now: Date = new Date()
): ScheduledUpdate;
```

`ScheduledUpdate` fields map 1:1 to `user_vocab_mastery` scalar columns so Plan 02 can `...spread` the result directly.
