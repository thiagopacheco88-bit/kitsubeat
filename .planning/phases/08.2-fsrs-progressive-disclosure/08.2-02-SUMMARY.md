# Phase 08.2 Plan 02: FSRS DB Writes — Server Action + Read Endpoints Summary

Per-answer FSRS persistence wired: `recordVocabAnswer` server action atomically writes one exercise log row and upserts one mastery row per call; two read endpoints expose tier state and mastery detail to the UI.

## What Was Built

### 1. `recordVocabAnswer` Server Action (`src/app/actions/exercises.ts`)

**Signature:**
```ts
export async function recordVocabAnswer(input: {
  userId: string;
  vocabItemId: string;            // target word UUID — NEVER pass distractor IDs
  songVersionId: string | null;   // null for kana-only exercises (Phase 9)
  exerciseType: ExerciseType;
  correct: boolean;
  revealedReading?: boolean;      // true if user tapped the reveal-reading hatch
  responseTimeMs: number;
}): Promise<{
  newTier: 1 | 2 | 3;
  newState: 0 | 1 | 2 | 3;
  reps: number;
  lapses: number;
  due: string;                    // ISO timestamp
}>
```

**Invariant (documented in JSDoc):** Distractor words receive NO mastery or log row. Caller MUST pass only the target `vocabItemId`.

**Transaction flow:**
1. `ratingFor(exerciseType, correct, { revealedReading })` → FSRS Rating
2. SELECT existing `user_vocab_mastery` row (null for first encounter)
3. `scheduleReview(prev, rating, intensity_preset)` → `ScheduledUpdate`
4. `insert(userVocabMastery).onConflictDoUpdate(...)` — upserts all FSRS scalar columns
5. `insert(userExerciseLog)` — one immutable log row per call
6. Returns `{ newTier, newState, reps, lapses, due }`

Throws if `vocabItemId` is empty (defensive guard for legacy data).

`saveSessionResults` is **unchanged** — song-level progress continues flowing through it.

### 2. GET /api/exercises/vocab-tiers — Batch Tier Loader

**URL:** `GET /api/exercises/vocab-tiers?ids=<uuid1>,<uuid2>,...&userId=<userId>`

**Limits:** max 200 IDs per request (400 if exceeded)

**Response:**
```json
{
  "tiers": {
    "<uuid1>": 1,
    "<uuid2>": 2,
    "<uuid3>": 3
  }
}
```

**Cold-start behavior:** Any requested ID without a `user_vocab_mastery` row defaults to `tierFor(0)` = Tier 1. This is the CONTEXT-locked decision: no row = new word.

**Cache-Control:** `private, no-store` — stale tabs re-fetch after background session updates.

### 3. GET /api/exercises/vocab-mastery/[vocabItemId] — Mastery Detail

**URL:** `GET /api/exercises/vocab-mastery/<uuid>?userId=<userId>`

**Response shape:**
```json
{
  "vocabItemId": "...",
  "state": 0,
  "tier": 1,
  "reps": 0,
  "lapses": 0,
  "correctPct": 0.0,
  "stability": null,
  "difficulty": null,
  "due": null,
  "lastReview": null,
  "totalAttempts": 0
}
```

**Missing mastery row:** Returns the "new word" synthesized shape above (state=0, tier=1, all counters 0, due=null). Never returns 404 on missing mastery — 404 only on malformed UUID route param.

`correctPct` is computed live from `user_exercise_log`: count(rating > 1) / count(*).

**Cache-Control:** `private, no-store`.
