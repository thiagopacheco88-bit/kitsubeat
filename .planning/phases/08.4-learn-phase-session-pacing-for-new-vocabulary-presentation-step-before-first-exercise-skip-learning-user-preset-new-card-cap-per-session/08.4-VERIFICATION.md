### Human Verification Required

#### 1. End-to-end cap filtering + JIT learn card flow

**Test:** Start a fresh exercise session for a song with more than 10 new words as a user with no subscription row.
**Expected:** Session builds with exactly 10 new/relearning words; remaining words are silently deferred. Each new word's first question is preceded by a LearnCard. Tapping the card background dismisses it to the QuestionCard at the same index (progress bar does not advance during the card).
**Why human:** Cap filtering + JIT card rendering + `currentIndex` invariant require a running dev server with live Neon DB state.

#### 2. Tap semantics: reveal vs. advance

**Test:** Tap "Show more" on a LearnCard to expand it, then tap the card background outside the revealed content.
**Expected:** First tap expands mnemonic + kanji_breakdown inline — card does not advance. Second tap on the background dismisses the LearnCard and shows the QuestionCard for the same question.
**Why human:** Multi-tap interaction model with `stopPropagation` boundaries requires browser rendering to confirm click-event routing.

#### 3. TTS speaker icon

**Test:** On a Chrome/Edge browser with a Japanese voice installed, tap the speaker icon on a LearnCard.
**Expected:** Japanese pronunciation plays. Card does not advance. Rapid double-tap cancels the first utterance and restarts.
**Why human:** `window.speechSynthesis` is browser-only; voice availability is device-dependent; `cancel()` behavior requires actual audio output.

#### 4. skip_learning toggle end-to-end

**Test:** Set "Skip learn cards" to checked on `/profile`, save. Then start a session for a song with new words.
**Expected:** No LearnCards render at all. Session goes straight to QuestionCard for every word including brand-new ones. New-word cap still applies.
**Why human:** Requires writing a DB pref row and verifying the session render path (need running app + DB).

#### 5. Free user profile UI

**Test:** Visit `/profile` as a user with no subscription row (or a canceled/free subscription).
**Expected:** "New words per session" input is visually disabled. "Upgrade to premium to raise the cap (up to 30)" hint text is visible. The skip_learning toggle still works.
**Why human:** UI disabled-state rendering and hint copy require visual inspection.

#### 6. Premium user cap editing

**Test:** Visit `/profile` as a user with an `active` + `premium_monthly` subscription row. Change the cap to 15 and save.
**Expected:** Cap input is enabled and accepts 1–30. After saving, reloading the page shows 15. Starting a new session respects the raised cap (up to 15 new words instead of 10).
**Why human:** Requires a live DB row with the correct plan/status values.

#### 7. Session resume across reload

**Test:** Start a session, see two LearnCards (dismiss both), then hard-reload the browser.
**Expected:** No LearnCards re-appear for the two already-dismissed words. Session resumes at the correct question.
**Why human:** `learnedVocabIds` localStorage persistence requires verifying actual browser storage behavior across reload.
